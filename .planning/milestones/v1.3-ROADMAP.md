# Milestone v1.3: Niche Discovery

**Status:** SHIPPED 2026-02-02
**Phases:** 15-18
**Total Plans:** 8

## Overview

Find high-potential topics with low competition that fit the channel's document-heavy format. Built complete niche discovery pipeline: demand analysis (Google Trends, autocomplete), competition classification (format, angles, quality), production constraints (animation detection, document scoring), and opportunity scoring (SAW formula with Channel DNA filtering).

## Phases

### Phase 15: Database Foundation & Demand Research

**Goal**: User can quantify demand for topics
**Depends on**: Phase 13 (keywords.db schema)
**Plans**: 2 plans

Plans:
- [x] 15-01: Schema extension + DemandAnalyzer foundation
- [x] 15-02: External data integration (trendspyg, scrapetube) + CLI

**Success Criteria:**
1. User can input a seed keyword and get search volume proxy (autocomplete position score)
2. User can see trend direction for a keyword (rising/stable/declining with percentage change)
3. User can expand a seed keyword into 10-20 related queries
4. User can see competition ratio score showing demand vs. supply balance

### Phase 16: Competition Analysis

**Goal**: User can identify gaps in existing coverage
**Depends on**: Phase 15 (demand data foundation)
**Plans**: 2 plans

Plans:
- [x] 16-01: Classifiers module + schema extension + database methods
- [x] 16-02: CompetitionAnalyzer extension + quality filtering + differentiation + CLI

**Success Criteria:**
1. User can see video count and unique channel count for a keyword
2. User can filter out low-quality content to see real competition
3. User can see what format competitors use (animation vs. documentary, political vs. legal angles)
4. User can see differentiation score identifying which angles are missing

### Phase 17: Format Filtering

**Goal**: Topics are filtered by production constraints
**Depends on**: Phase 16 (competition data)
**Plans**: 2 plans

Plans:
- [x] 17-01: Production constraint foundation (animation detection + document scoring)
- [x] 17-02: Source hints + CLI for production constraint evaluation

**Success Criteria:**
1. Topics requiring animation are flagged as hard blocks before research investment
2. Topics are scored 0-4 for document-friendliness (treaty-heavy = 4, concept-heavy = 0)
3. User can verify academic source availability for a topic before committing

### Phase 18: Opportunity Scoring & Orchestrator

**Goal**: User gets ranked topic opportunities with full context
**Depends on**: Phases 15, 16, 17 (all data layers)
**Plans**: 2 plans

Plans:
- [x] 18-01: OpportunityScorer + lifecycle state tracking + Channel DNA filtering
- [x] 18-02: OpportunityOrchestrator + report generation + CLI

**Success Criteria:**
1. User can see combined opportunity score (demand x gap x fit / effort) for any topic
2. Production constraints are weighted in scoring (animation-required topics score 0 regardless of demand)
3. Channel DNA rules auto-reject topics (clickbait language, news-first framing automatically filtered)
4. Opportunities track lifecycle status from DISCOVERED through PUBLISHED
5. User can generate Markdown opportunity report with all decision factors documented

## Milestone Summary

**Key Decisions:**

| Decision | Rationale | Outcome |
|----------|-----------|---------|
| 7-day cache default | Demand data changes; balance freshness vs API calls | Good |
| Conservative opportunity thresholds (4x+) | High research overhead means only obvious wins | Good |
| Channel-based format detection | Channel name is stronger signal than title keywords | Good |
| Multi-angle support (JSON arrays) | Real videos combine perspectives (legal + historical) | Good |
| SAW formula with equal weights | Simple, transparent, configurable for future tuning | Good |
| Hard constraint pre-filtering | Return None for blocked topics, not 0 | Good |
| Channel DNA violations advisory | Advisory warnings, not blocking (user decides) | Good |

**Issues Resolved:**
- External packages (trendspyg, scrapetube) optional — graceful degradation when not installed
- Unicode arrows may cause Windows cp1252 issues — ASCII fallback available
- Schema migration automatic via _ensure_*_columns() methods

**Technical Debt Incurred:**
- External package dependencies need documentation for optional features
- Voice patterns still require `--rebuild-voice` manual setup (from v1.2)
- datetime.utcnow() deprecation warning in intent_mapper.py (from v1.2)
