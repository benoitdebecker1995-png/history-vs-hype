  <name>PacingChecker — core pacing analysis engine</name>
  <files>tools/script-checkers/checkers/pacing.py, tools/script-checkers/tests/test_pacing.py</files>
  <behavior>
    PacingChecker inherits BaseChecker and implements check(text) -> dict.

    **Section parsing:** Uses ScriptParser from tools/production/parser.py (import via sys.path or relative path — match how cli.py runs from tools/script-checkers/ directory). parse_text(text) returns List[Section] with heading, content, word_count, section_type.

    **Import path note:** cli.py runs from tools/script-checkers/ so sys.path needs tools/ parent added to import production.parser. Follow same pattern as existing checkers. Add parent directory to sys.path at top of pacing.py: `sys.path.insert(0, str(Path(__file__).resolve().parent.parent.parent))` to enable `from tools.production.parser import ScriptParser`.

    **Metric calculations (per section):**

    1. **Sentence variance (PACE-01):**
       - Use spaCy (lazy-loaded) to split section content into sentences
       - Count words per sentence (exclude punctuation and spaces)
       - Calculate standard deviation via statistics.stdev()
       - Flag if stdev > 15 (from config)
       - If fewer than 2 sentences, variance = 0.0
       - Strip B-roll markers from content before analysis (use ScriptParser's marker stripping approach — strip all bracketed markers `[...]` from text before NLP processing)

    2. **Flesch Reading Ease delta (PACE-02):**
       - Calculate Flesch score per section using textstat.flesch_reading_ease() (lazy import)
       - Delta = current section score minus previous section score
       - Flag if absolute delta > 20 (from config)
       - First section has delta = 0 (no previous)
       - Strip B-roll markers before calculating

    3. **Entity density (PACE-03):**
       - Use spaCy POS tagging: count tokens with pos_ == "PROPN"
       - Divide by total non-punctuation non-space tokens
       - Flag if ratio > 0.4 (from config)
       - If zero total words, density = 0.0
       - Strip B-roll markers before analyzing

    4. **Composite score (PACE-04):**
       - Score per section on 0-100 scale (100 = perfect, 0 = worst)
       - Start at 100, deduct points per metric:
         - Sentence variance penalty: min(30, (variance - threshold) * 2) if variance > threshold, else 0
         - Flesch delta penalty: min(35, (abs(delta) - threshold) * 1.75) if abs(delta) > threshold, else 0
         - Entity density penalty: min(35, (density - threshold) * 87.5) if density > threshold, else 0
       - Score = max(0, 100 - total_penalties)
       - Penalties are capped so no single metric can tank the score below the others' range

    5. **Energy arc sparkline (PACE-06):**
       - generate_sparkline(scores: List[float]) -> str
       - Uses Unicode block characters: "▁▂▃▄▅▆▇█"
       - Map scores to 0-7 index via min-max normalization
       - Edge cases: empty list -> "", all identical -> "▄" * len, single score -> "▄"
       - NOTE: Sparkline maps COMPLEXITY (inverted score), not score directly. Low score = high complexity = tall bar. So invert: complexity = 100 - score. This makes the sparkline show "energy" — tall bars mean complex sections.

    6. **Flat zone detection (PACE-06):**
       - detect_flat_zones(scores: List[float]) -> List[str]
       - 3+ consecutive sections within 10 points of each other = flat zone
       - Returns list of advisory strings naming the section range
       - Fewer than 3 sections -> no flat zones possible

    7. **Hook/interrupt detection (PACE-05):**
       - Advisory only — NOT included in composite score
       - detect_hooks(text: str) -> List[int]: Find character positions of modern relevance markers
         - Time keywords: today, 2024, 2025, 2026, still, currently, now (word-boundary, case-insensitive)
         - B-roll markers: [NEWS ...], [MODERN ...], [CURRENT ...]
       - check_hook_gaps(sections: List[Section]) -> List[str]: For each section, check if any hooks present. Sections without hooks get advisory note.
       - Pattern interrupt detection: Detect B-roll marker variety within sections. Sections >200 words with zero B-roll markers get "consider adding visual variety" advisory.

    **Verdict (top-level):**
    - Calculate average of all section scores
    - PASS: average >= 75
    - NEEDS WORK: average >= 50 and < 75
    - FAIL: average < 50
    - SKIPPED: single-section script (no multi-section analysis possible)

    **Return format:**
    ```python
    {
        'issues': [  # Only sections with score < 75 (problems-only default)
            {
                'section': 'Section Name',
                'score': 62,
                'metrics': {

                ]
            }
        ],
        'all_sections': [  # Full data for verbose mode
            {
                'section': 'Section Name',
                'score': 62,
                'metrics': { ... },
                'reasons': [ ... ]
            }
        ],
        'stats': {
            'total_sections': 8,
            'flagged_sections': 2,
            'average_score': 72,
            'energy_arc': '▅▃▆▇▂▄▅▃',
            'flat_zones': ['Energy plateau: sections 3-5 have similar complexity — consider pattern interrupt'],
            'verdict': 'NEEDS WORK'
        },
        'advisories': [  # Hook/interrupt advisories (separate from score)
            'Section "The Ottoman Response" (420 words) has no modern relevance hooks',
            'Section "Treaty Details" has no B-roll markers — consider adding visual variety'
        ]
    }
    ```

    RED tests to write first:

    1. `test_sentence_variance_below_threshold` — text with uniform sentence lengths -> variance < 15, no flag

    3. `test_sentence_variance_single_sentence` — one sentence -> variance = 0.0
    4. `test_flesch_delta_within_threshold` — two sections with similar complexity -> delta < 20, no flag
    5. `test_flesch_delta_exceeds_threshold` — simple section followed by complex section -> delta > 20, flagged
    6. `test_flesch_delta_first_section` — first section has delta = 0

    9. `test_entity_density_empty_text` — empty string -> density = 0.0
    10. `test_composite_score_perfect` — all metrics below thresholds -> score = 100
    11. `test_composite_score_degraded` — one metric over threshold -> score < 100
    12. `test_composite_score_floor` — all metrics very bad -> score >= 0
    13. `test_sparkline_generation` — known scores -> expected sparkline chars
    14. `test_sparkline_all_identical` — all same score -> all mid-level bars
    15. `test_sparkline_empty` — empty list -> empty string
    16. `test_flat_zone_detected` — 4 sections with similar scores -> 1 flat zone
    17. `test_flat_zone_none` — varied scores -> 0 flat zones

    19. `test_hook_detection_finds_keywords` — text with "today" and "currently" -> positions found

    21. `test_check_single_section_skipped` — single-section script -> verdict = SKIPPED
