**Part B: Standardize error exits across youtube_analytics/** (CLI-02)

For ALL files in youtube_analytics/ that have CLI entry points, verify/add this pattern in __main__:

```python
result = some_function(...)
if isinstance(result, dict) and "error" in result:
    print(f"ERROR: {result['error']}", file=sys.stderr)
    sys.exit(1)
```

Check for these anti-patterns and FIX:
- `print(f"Error: {msg}")` without `file=sys.stderr` → add `file=sys.stderr`
- Error without `sys.exit(1)` → add sys.exit(1)
- `sys.exit(0)` on error → change to sys.exit(1)
- `return` after error in __main__ without exit code → add sys.exit(1)

Do NOT change error dict returns in library functions — those are correct (Phase 50 established the pattern). Only standardize how CLI entry points HANDLE those error dicts.
