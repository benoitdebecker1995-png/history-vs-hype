  <action>
Create `tools/logging_config.py` with:

1. `setup_logging(verbose: bool = False, quiet: bool = False) -> None`:
   - Configure `logging.getLogger("tools")` (NOT the root Python logger)
   - quiet → ERROR level, default → INFO level, verbose → DEBUG level
   - Clear existing handlers before adding new one (root.handlers.clear())
   - StreamHandler pointing to sys.stderr
   - Default format: "%(levelname)s: %(message)s"
   - Verbose format: "[%(name)s] %(levelname)s: %(message)s"
   - Color support: check `sys.stderr.isatty()`, if True use _ColorFormatter with colorama.init()
   - Colors: DEBUG=cyan(\033[36m), INFO=green(\033[32m), WARNING=yellow(\033[33m), ERROR=red(\033[31m), CRITICAL=bold-red(\033[31;1m)
   - Set root.propagate = False
   - Raise ValueError if both verbose and quiet are True (defensive — argparse prevents this, but library callers might not)

2. `get_logger(name: str) -> logging.Logger`:
   - Simple wrapper: `return logging.getLogger(name)`
   - Docstring showing usage: `logger = get_logger(__name__)`

3. `_ColorFormatter(logging.Formatter)`:
   - Override format() to add ANSI color codes around levelname
   - Use logging.makeLogRecord(record.__dict__) to avoid mutating the original record

4. Helper functions `_default_fmt()` and `_verbose_fmt()` returning format strings

Update `tools/__init__.py`:
- Add `logging.getLogger("tools").addHandler(logging.NullHandler())` for library safety
- Keep any existing content

Do NOT use logging.basicConfig(). Do NOT configure the root Python logger.

```python
verbosity = parser.add_mutually_exclusive_group()

verbosity.add_argument("--quiet", "-q", action="store_true", help="Only show errors on stderr")
```

Then after `args = parser.parse_args()`, add:

```python

```

Do NOT pass verbose/quiet as parameters to library functions. The logger hierarchy handles verbosity automatically.
