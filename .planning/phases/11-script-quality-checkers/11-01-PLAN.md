**stumble.py (SCRIPT-03):**
- `StumbleChecker(BaseChecker)` class
- Uses spaCy for sentence parsing and dependency analysis
- Detection logic from RESEARCH.md:
  - Flag sentences with word_count > config.stumble_max_words
  - Flag sentences with 2+ subordinate clauses (dep_ in ['advcl', 'acl', 'relcl'])
  - Severity: 'medium' for 25-30 words OR 2 clauses, 'high' for >30 words OR 3+ clauses
- Issue format: `{line: N, text: "...", word_count: N, sub_clauses: N, severity: 'medium'|'high', reason: "..."}`
- Load spaCy model lazily (on first check call) to avoid import-time overhead

**scaffolding.py (SCRIPT-04):**
- `ScaffoldingChecker(BaseChecker)` class
- Uses regex for pattern matching
- Detection logic:
  - Match scaffolding phrases from config (case-insensitive)
  - For "so" and "now": only flag if sentence-initial (after period/newline)
  - Calculate threshold using `calculate_threshold(word_count, config.scaffolding_base_rate)`
  - Severity based on count vs threshold:
    - count <= threshold: 'ok'
    - count <= threshold * warn_multiplier: 'warning'
    - count > threshold * error_multiplier: 'error'
- Issue format: `{phrase: "here's", count: N, positions: [char_indices], threshold: N, severity: 'ok'|'warning'|'error'}`
- Stats: total_scaffolding, words, threshold, rate_per_1000

Special handling per RESEARCH.md Pitfall 5:
- "Here's what [X] actually says" is a SIGNATURE PHRASE, not filler
- Detect this pattern and mark as 'rhetorical' (don't count toward limit)

Handle errors gracefully:
