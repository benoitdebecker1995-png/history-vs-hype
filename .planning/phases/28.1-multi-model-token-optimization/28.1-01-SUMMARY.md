## Decisions Made

### 1. Routing Strategy: Manual Over Automated
**Decision:** Use native Claude Code `/model` command for manual model switching, not automated routing.

**Rationale:**
- 68 total invocations/month = ~2-3 per day
- Manual switching takes 2 seconds: `/model openrouter/...`
- Automated routing adds: config maintenance, error handling, debugging time
- Tradeoff: Save 2.3 minutes/month automation time vs config complexity and loss of control
- Verdict: Not worth automation for this usage level

### 2. OpenRouter Free Tier Only
**Decision:** Use OpenRouter free tier (50 req/day) without adding $10 credits initially.

**Rationale:**
- Current routable task usage: ~2 req/day
- Free tier capacity: 50 req/day
- Headroom: 25x current usage
- Growth scenario: Even at 8 videos/month (2x current), only 4 req/day needed
- Verdict: Free tier sufficient for foreseeable usage

### 3. Local Models Rejected
**Decision:** Do not pursue Ollama or local model routing.

**Rationale:**
- Total RAM: 14.9GB (below Qwen 32B requirement of 20-24GB)
- Available RAM: 3.7GB (7B models need 4-6GB, would risk system instability)
- GPU: AMD Radeon integrated (not NVIDIA CUDA, limited model support)
- Verdict: Hardware insufficient for useful local models, focus on OpenRouter

### 4. Opus Tasks Stay on Claude
**Decision:** Keep `/script` and `script-writer-v2` on Claude Opus (do not route).

**Rationale:**
- These are the channel's highest-value outputs (84.9% of costs, $473.51/month)
- Require retention engineering, style matching (Kraut/Alex O'Connor patterns)
- Free models cannot match quality for 8,000-word academic scripts
- Savings would be offset by quality degradation and user editing time
- Verdict: Quality-critical tasks not worth routing

### 5. Routable Task Criteria
**Decision:** Only route Haiku-tier mechanical tasks (10 of 19 total tasks).

**Rationale:**
- Haiku tasks already near-zero cost ($0.71/month), routing removes friction not cost
- Sonnet tasks need reasoning/synthesis (source evaluation, pattern analysis)
- Routing criteria: Template-based, no creativity, deterministic output
- Tasks: /status, /help, /fix, /sources, /prep, /discover + 3 agents
- Verdict: Conservative routing preserves quality while testing free models
