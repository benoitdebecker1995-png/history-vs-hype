  <action>
Create `tools/youtube-analytics/backfill.py` with the following components:

**1. `import_from_json_prefetch(project_root: Path) -> dict`**
- Load `_longform_enriched.json` (preferred, has CTR + retention) or fall back to `_longform_metrics.json`
- For each video: print per-video progress "Importing {title_short} ({i}/{total})... done" (per locked decision)
- Upsert into `video_performance` via `KeywordDB.add_video_performance()`
- Compute conversion_rate from views and subscribers_gained
- Return `{'imported': N, 'skipped': N, 'errors': [...]}`

**2. `import_from_analysis_files(project_root: Path, force: bool) -> dict`**
- Call existing `feedback_parser.backfill_all(project_root, force=force)`
- This handles scanning all POST-PUBLISH-ANALYSIS files from both project folders and channel-data/analyses/
- Per RESEARCH pitfall 4: analysis files with "Performance data unavailable" are low-quality — don't let null values overwrite good API data. After backfill_all(), do NOT re-upsert metrics that are None.
- Return the result from backfill_all()

**3. `reclassify_topics(project_root: Path) -> int`**
- Query all rows from video_performance where topic_type is 'general' or NULL
- For each, re-run classification with an EXPANDED vocabulary that includes:
  - The existing TAG_VOCABULARY from performance.py
  - Additional patterns: title contains 'constitution' or 'democratic' → 'legal', title contains 'myth' or 'dark ages' or 'flat earth' → 'ideological', title contains 'fact-check' or 'claims' → 'factcheck'
- Use `classify_topic_type(title)` from performance.py as base, then apply expanded rules if still 'general'
- Update the database rows with corrected topic_type
- Return count of reclassified rows

**4. `generate_channel_insights_report(project_root: Path) -> dict`**
- CRITICAL: Filter to own-channel videos only. Use video IDs from JSON pre-fetches as the authoritative list of own-channel videos. Do NOT include competitor videos.
- Query video_performance for own-channel video IDs only
- Build composite scores: weighted blend of views (0.4), avg_retention (0.35), subscriber conversion (0.25) — normalize each metric 0-1 against channel max
- Generate `channel-data/channel-insights.md` with these sections:
  - Header with generation timestamp and video count
  - "Auto-generated. Do not edit manually." notice
  - **Performance by Topic Type** table: topic, videos, avg views, avg retention, avg conversion, signal level
  - **Top Performers** list: best 3-5 videos by composite score with reasoning ("similar to your Essequibo video (23K views, 19 subs)")
  - **Channel Signals** section: advisory insights with confidence flagging
    - 1-2 videos: "early signal"
    - 3-5 videos: "moderate signal"
    - 6+ videos: "strong signal"
  - **Topic Recommendations** section: boost proven patterns but also surface novel/underexplored opportunities
  - Footer: "Experiment freely — these are patterns, not prescriptions."
- Per locked decision: insights are advisory, not prescriptive
- Return `{'saved_to': path, 'video_count': N}` or `{'error': msg}`

**5. `run_backfill(project_root: Path, force: bool = False) -> dict`**
- Orchestrates: Stage 1 (JSON import) → Stage 2 (markdown parsing) → Stage 3 (reclassification) → Stage 4 (insights generation)
- Prints section headers for each stage
- Collects results from each stage
- Returns combined results dict

**6. CLI (`if __name__ == '__main__'`)**
- argparse with flags: `--force` (re-import), `--insights-only` (skip import), `--json-only` (skip markdown)
- project_root = Path(__file__).parent.parent.parent
- Print summary after completion

- Do NOT re-fetch from YouTube API. Use JSON pre-fetches as primary source
- Do NOT include competitor videos in insights (filter by known own-channel IDs from JSONs)
- Do NOT make insights prescriptive — always advisory language
