<task type="auto">
  <name>Task 1: Add Research Verification Gate (GATE-01, GATE-02)</name>
  <files>.claude/commands/script.md</files>
  <action>
Add a new section titled "## Research Verification Gate (MANDATORY — Runs Before Script Generation)" to `.claude/commands/script.md`. Insert it IMMEDIATELY BEFORE the "## Duration & Structure Gate" section (around line 147), so it runs before any writing begins.

The section must instruct Claude to:

1. **Locate the project's `01-VERIFIED-RESEARCH.md`** — Glob for `video-projects/**/[project]/01-VERIFIED-RESEARCH.md`. If not found, emit a warning ("No verified research file found — skipping verification gate") and proceed (do NOT block on missing file).

2. **Count markers** — Scan the file content for these status markers:
   - `✅` (or "VERIFIED") = verified claim
   - `⏳` (or "RESEARCHING") = pending claim
   - `❌` (or "UNVERIFIABLE") = failed claim
   - Count total = verified + pending + failed
   - Calculate percentage: `verified / total * 100`

3. **Gate logic:**
   - If percentage < 90%: **BLOCK**. Display:
     ```
     --- RESEARCH VERIFICATION GATE: BLOCKED ---
     Project: [project-name]
     Verified: X/Y claims (Z%)
     Pending: N claims (⏳)
     Failed: M claims (❌)

     Cannot proceed: Research must be >=90% verified before scripting.
     Fix: Complete verification in 01-VERIFIED-RESEARCH.md, then re-run /script.
     ---
     ```
     Then STOP. Do not proceed to script generation.

   - If percentage >= 90%: **PASS**. Display:
     ```
     --- Research Verification Gate: PASSED ---
     Verified: X/Y claims (Z%)
     [If any pending/failed: "Note: N claims still pending/failed — verify before filming"]
     ---
     ```
     Then proceed normally to Duration & Structure Gate and script generation.

4. **Skip conditions** — This gate only applies to `--new` and default (interactive write) modes. Skip for `--revise`, `--review`, `--teleprompter`, `--hooks`, `--collaborate` (these work on existing scripts, not new generation from research).

Write the section as prose instructions for Claude (matching the style of other sections in script.md like "Duration & Structure Gate" and "YouTube Intelligence Check"), NOT as Python code. The marker counting is simple text scanning that Claude does natively when reading the file.
  </action>
  <verify>
    <automated>grep -c "Research Verification Gate" "D:/History vs Hype/.claude/commands/script.md"</automated>
  </verify>
  <done>The script.md file contains a Research Verification Gate section that: (1) blocks with count/percentage when <90% verified, (2) shows summary and proceeds when >=90%, (3) only applies to --new and default modes, (4) is positioned before the Duration & Structure Gate</done>
