Create `.claude/commands/translate.md` following the established slash command pattern (see research.md, verify.md for examples).

**Structure:**
```
# /translate - Document Translation Pipeline

Translate legal/historical documents clause-by-clause with cross-checking, legal annotations, and surprise detection.

## Usage
/translate [project-folder] --file [document-path] --language [source-language]
/translate [project-folder] --file [document-path] --language [source-language] --narrative "expected narrative"
```

**Flags section:**
| Flag | Purpose | Example |
|------|---------|---------|
| `--file` | Path to document file | `--file treaty.txt` |
| `--language` | Source language | `--language french` |
| `--narrative` | Expected narrative for surprise detection | `--narrative "Treaty establishes border at river"` |
| `--skip-crosscheck` | Skip cross-check step | |
| `--skip-annotations` | Skip legal annotation step | |

**Workflow section (the core — this IS the orchestration prompt):**

## TRANSLATE WORKFLOW

### Step 1: Detect Document Structure
- Run: `python tools/translation/cli.py detect --file [document-path]`
- Report to user: "Detected N sections: [section list]"
- If detection fails: HALT and report error to user

### Step 2: Translate Each Clause (Claude Code Native)
For each detected section/clause:
1. Run Python to build payload:
   ```
   python -c "
   import sys; sys.path.insert(0, 'tools/translation')
   from translator import TranslationDataBuilder
   import json
   builder = TranslationDataBuilder()
   payload = builder.build_translation_payload(clause_text, full_document, source_language, clause_id, document_context)
   print(json.dumps(payload))
   "
   ```
2. Use the returned system_prompt and user_prompt to call Claude natively (model: claude-sonnet-4-6)
3. Parse the response:
   ```
   python -c "
   import sys; sys.path.insert(0, 'tools/translation')
   from translator import TranslationDataBuilder
   import json
   builder = TranslationDataBuilder()
   result = builder.parse_response(response_text, clause_id, original_text)
   print(json.dumps(result))
   "
   ```
4. Report progress: "Translated clause N/M ([clause_id])"

### Step 3: Cross-Check (if not --skip-crosscheck)
- For each clause: get independent translation from DeepL/googletrans backend
- Compare semantically using Claude Code natively (model: claude-sonnet-4-6)
- If severity='significant': **HALT AND ASK USER**
  - Display the discrepancy clearly
  - Options: (1) Accept Claude's translation (2) Use alternative (3) Pause for manual review
  - Wait for user response before continuing

### Step 4: Legal Annotation (if not --skip-annotations)
- For each section: annotate legal terms using Claude Code natively (model: claude-sonnet-4-6)
- Parse annotations and attach to translation results

### Step 5: Surprise Detection (if --narrative provided)
- For each section: detect surprising clauses using Claude Code natively (model: claude-sonnet-4-6)
- Flag HIGH severity surprises to user

### Step 6: Format Output
- Run: `python tools/translation/cli.py detect ...` (structure info) + formatter
- Write formatted output to `[project-folder]/[document-name]-TRANSLATION-FORMATTED.md`
- Report: "Translation complete! Output saved to [path]"

**On pipeline failure at any step:** HALT and report the error to user. Do not auto-continue. Ask: "Step N failed: [error]. Would you like to (1) retry this step, (2) skip and continue, or (3) abort?"

**Model selection per step:**
| Step | Model | Reasoning |
|------|-------|-----------|
| Translation | claude-sonnet-4-6 | Nuanced legal/historical language |
| Cross-check comparison | claude-sonnet-4-6 | Semantic comparison accuracy |
| Legal annotation | claude-sonnet-4-6 | Legal terminology expertise |
| Surprise detection | claude-sonnet-4-6 | Contextual reasoning about narratives |

**After completion:**
> "Translation pipeline complete!
> Output: [path to formatted file]
> Next steps:
> - Review the formatted translation
> - Run `/verify --translation [project]` for verification
> - When ready: `/script --document-mode` to write the script"

**Notes section:**
- This command replaces the standalone `python cli.py translate` workflow
- All LLM calls are made by Claude Code natively — no API key needed
- One document at a time (no batch mode)
- Cross-check uses DeepL/googletrans for independent translation + Claude for semantic comparison
