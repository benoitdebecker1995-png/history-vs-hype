# Phase 42.1 Plan 02: Remaining 4 Modules Refactor + CLI + Command Updates Summary

**One-liner:** Applied the prompt extraction pattern (validated in Plan 01) to all 4 remaining translation modules, updated cli.py to redirect LLM commands to /translate, and updated verify.md and translate.md with correct class names and no-API-key interfaces.

---

## What Was Built

### cross_checker.py (refactored)

Removed `import anthropic` and `from env_loader import ...`. The `_translate_with_backend()` method (DeepL/googletrans backend calls) is retained unchanged — these call independent translation services, not Anthropic.

New methods replacing the old `_compare_translations()` Anthropic call:
- `build_comparison_payload(claude_translation, backend_translation, original_text, clause_id, source_language, backend)` — assembles `{clause_id, system_prompt, user_prompt}` for Claude Code to execute
- `parse_comparison_response(response_text, clause_id)` — extracts JSON from Claude response, returns `{clause_id, has_discrepancy, severity, explanation, recommendation}`

The class docstring updated to: "Pure data processor. LLM calls handled by Claude Code via /translate command."

### legal_annotator.py (refactored)

Removed `import anthropic` and `from env_loader import ...`. The `LegalAnnotator` class is now a pure data processor:
- `build_annotation_payload(clause_text, translation, clause_id, source_language, document_context)` — returns `{clause_id, system_prompt, user_prompt}` for Claude Code to execute
- `parse_annotation_response(response_text, clause_id)` — parses Claude response, returns `{clause_id, annotations, footnotes}`
- `format_footnotes(annotations)` — formatting helper retained unchanged

### surprise_detector.py (refactored)

Removed `import anthropic` and `from env_loader import ...`. The `SurpriseDetector` class is now a pure data processor:
- `build_surprise_payload(clause_text, translation, narrative_baseline, clause_id, source_language, document_context, full_context)` — returns `{clause_id, system_prompt, user_prompt}` for Claude Code to execute
- `parse_surprise_response(response_text, clause_id, original, translation)` — parses Claude response, returns structured surprise result with severity, explanation, what_people_think, what_document_says, script_beat
- `format_report(results)`, `_format_surprise_entry(surprise)`, `update_sections_with_surprises(sections, surprises)` — all formatting/merging helpers retained unchanged

### verification.py (refactored)

Removed `import anthropic` and `from env_loader import ...`. The `TranslationVerifier` class is restructured:
- `__init__(project_dir=None)` — takes no API-related arguments; just optional project directory
- `_audit_existing_output(content, file_path)` — UNCHANGED, pure Python regex parsing, no LLM
- `verify_translation(translation_file, mode='audit', scholarly_result=None)` — accepts pre-computed scholarly result from Claude Code
- `build_scholarly_comparison_payload(translation_text, scholarly_summary, document_name)` — returns `{system_prompt, user_prompt}` for Claude Code to execute
- `parse_scholarly_comparison_response(response_text)` — parses scholarly comparison response
- `build_knowledge_comparison_payload(translation_text, document_name)` — returns payload for knowledge comparison
- `parse_knowledge_comparison_response(response_text)` — parses knowledge comparison response

Audit mode requires zero changes and zero API calls. `_run_fresh_verification()` removed (was Anthropic-dependent; Claude Code now handles this via /verify command flow).

### cli.py (updated)

All LLM-dependent subcommands (`translate`, `crosscheck`, `annotate`, `surprise`) now redirect to /translate with a clear error message and `sys.exit(1)`. The `detect` and `smoketest` subcommands remain fully functional without any API key.

Module docstring updated to document the two tiers: standalone (detect, smoketest) and replaced-by-slash-command (translate, crosscheck, annotate, surprise).

### verify.md (updated)

Updated the `## TRANSLATION VERIFICATION` section:
- Removed `--fresh` flag (was Anthropic-dependent)
- Added `--document-name` flag for Claude knowledge comparison
- Documented that audit mode is pure Python with no API key
- Documented payload builder pattern: `TranslationVerifier.build_scholarly_comparison_payload()` builds prompt, Claude Code executes LLM call natively, `parse_scholarly_comparison_response()` parses result
- Updated `TranslationVerifier()` constructor usage (no model argument)

### translate.md (updated)

Updated Steps 3, 4, and 5 to use proper payload builder/parser patterns for all refactored classes:
- **Step 3 cross-check:** `CrossChecker.build_comparison_payload()` + `parse_comparison_response()`
- **Step 4 annotation:** `LegalAnnotator.build_annotation_payload()` + `parse_annotation_response()`
- **Step 5 surprise:** `SurpriseDetector.build_surprise_payload()` + `parse_surprise_response()`
