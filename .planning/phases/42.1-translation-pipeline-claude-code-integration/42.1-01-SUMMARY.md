# Phase 42.1 Plan 01: Translator.py Refactor + /translate Command Summary

**One-liner:** Refactored `translator.py` to pure data processor (`TranslationDataBuilder`) with prompt builder and response parser, deleted `env_loader.py`, and created `/translate` slash command that orchestrates the full translation pipeline via Claude Code native LLM calls.

---

## What Was Built

### translator.py (refactored)

`Translator` class renamed to `TranslationDataBuilder`. All Anthropic SDK imports and API client initialization removed. The class is now a pure data processor with zero LLM calls:

- `build_translation_payload(clause_text, full_document, source_language, clause_id, document_context)` — assembles `{clause_id, original, system_prompt, user_prompt}` payload for Claude Code to execute
- `parse_response(response_text, clause_id, original_text)` — converts raw Claude response into `{clause_id, original, translation, notes}` structured dict
- All pure Python helpers retained unchanged: `_highlight_clause`, `_build_translation_prompt`, `_parse_translation_response`
- `TRANSLATION_SYSTEM_PROMPT` constant retained in Python (domain knowledge ownership)

### env_loader.py (deleted)

Removed entirely. No longer needed — the refactored translation pipeline uses Claude Code's native execution context instead of direct Anthropic API calls.

### smoke_test.py (rewritten)

Replaced 5-step credential+API smoke test with 4-step pure Python health check:
1. Module health check — verifies `TranslationDataBuilder`, `StructureDetector`, `Formatter` all import and instantiate correctly
2. Structure detection — validates structure detection on test French document
3. Payload builder — verifies `build_translation_payload()` returns correct dict structure
4. Response parser — verifies `parse_response()` correctly extracts TRANSLATION: and NOTES: from mock Claude response

All 4 steps pass in 0.0s with no API calls.

### /translate slash command (created)

`.claude/commands/translate.md` — orchestrates the full 6-step translation pipeline:
1. Detect document structure via `cli.py detect`
2. Translate each clause: Python builds payload → Claude Code calls LLM natively → Python parses response
3. Cross-check: DeepL/googletrans backend + Claude semantic comparison
4. Legal annotation via Claude Code native calls
5. Surprise detection vs. expected narrative
6. Format and save to `[project]/[document]-TRANSLATION-FORMATTED.md`

Key behaviors:
- All LLM calls use `claude-sonnet-4-6` (no API key required)
- Halt-and-ask on any step failure
- Halt-and-ask on significant cross-check discrepancies (severity = 'significant')
- One document at a time (no batch mode)
