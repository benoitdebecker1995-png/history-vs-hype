# Phase 42.1: Translation Pipeline Claude Code Integration - Context

**Gathered:** 2026-02-20
**Status:** Ready for planning

<domain>
## Phase Boundary

Refactor the 5 translation pipeline modules (translator, cross_checker, legal_annotator, surprise_detector, verification) to route LLM calls through Claude Code instead of direct Anthropic API calls. Eliminates the need for separate API keys — Pro plan users get translation functionality through their existing Claude Code subscription. This is a quality-of-life refactor, not a feature addition.

</domain>

<decisions>
## Implementation Decisions

### LLM Routing Approach
- Claude picks the architecture (agent-mediated vs prompt passthrough vs hybrid)
- Roll out incrementally: refactor one module first (proof of concept), then apply pattern to remaining 4
- Remove the direct API path entirely — clean break, no dual code paths
- Claude decides on smoke test fate (keep/adapt or remove based on whether it still adds value)
- No special rate-limit handling — let Claude Code manage its own limits natively
- Keep it one document at a time — no batch mode

### Slash Command Boundary
- Claude decides the split between Python data processing and Claude Code LLM orchestration
- Delete env_loader.py and all API key management code — no longer needed
- Check for other dependents of env_loader.py / anthropic SDK outside translation (unknown — researcher should investigate)
- Claude decides whether standalone Python CLI access (`python cli.py translate`) is kept or removed
- Claude decides on file output format (same artifacts or simplified)
- On pipeline failure mid-run: halt and ask the user what to do (don't auto-continue)
- The pipeline itself works fine — the only pain point is the API key requirement

### Model Selection
- Hardcode model per pipeline step (optimized per task complexity)
- Claude assigns which model per step based on quality needs
- Use latest model aliases (e.g., `claude-sonnet-4-6`), not pinned date versions
- Claude decides whether a per-run model override flag adds value
- No rate-limit awareness needed — Claude Code handles this

### Claude's Discretion
- Overall LLM routing architecture (how Python tools communicate with Claude Code)
- Prompt ownership (Python vs agent layer)
- Pipeline parallelization opportunities
- Output verbosity (intermediate steps shown or just final result)
- Standalone CLI access retention
- File output format changes
- Smoke test retention
- Per-run model override flag
- Additional output validation layer

</decisions>
