Create `tools/document_discovery/` package with three files.

**`__init__.py`**: Package init, version string, convenience imports.

**`gap_checker.py`** — `GapChecker` class (DISC-01):
- `check_gap(query: str) -> Dict[str, Any]`: Main entry. Accepts free-form document description (e.g., "Statut des Juifs 1940", "Bartolome de las Casas Brevisima relacion"). Language-agnostic from the start.
- Constructs search queries for each source category and returns structured results:
  1. **Academic sourcebooks**: Search patterns for major sourcebook series (e.g., "Medieval Sourcebook", "Internet Sourcebook Project", academic anthologies). Generate search URLs for Google Scholar with queries like `"{document name}" English translation site:*.edu`.
  2. **Google Scholar + JSTOR**: Search for `"{document}" English translation` and `"{document}" translated`. Generate Scholar and JSTOR search URLs.
  3. **Government/institutional**: For legal documents, check government translation portals (e.g., Legifrance English, EUR-Lex). Generate appropriate search URLs.
  4. **General web**: Generate Google search URLs for `"{document}" full English translation`.
- For each category, return: `{'category': str, 'search_url': str, 'description': str}` — these are search URLs the user opens to verify, NOT automated web scraping.
- Overall result dict: `{'query': str, 'searches': List[Dict], 'qualification_criteria': str, 'instructions': str}`
- Qualification criteria text: document qualifies if (a) no full English translation exists, OR (b) existing translations are misleading/distort the original. Partial translations or summaries do not disqualify.
- Return `{'error': msg}` on failure (error dict pattern).

**`archive_lookup.py`** — `ArchiveLookup` class (DISC-03):
- `lookup(query: str, language: Optional[str] = None) -> Dict[str, Any]`: Main entry. Accepts document description + optional language hint.
- Build search URLs for each archive, always leading with academic/critical editions:
  1. **Academic editions**: Google Scholar search for critical editions with ISBN. Generate URL: `scholar.google.com/scholar?q="{document}" critical edition`
  2. **Legifrance** (French law): `legifrance.gouv.fr` search URL
  3. **Gallica (BnF)**: `gallica.bnf.fr` search URL (French national library)
  4. **Wikisource variants**: Generate URLs for `{lang}.wikisource.org/wiki/Special:Search?search={query}` — construct for French, Spanish, German, Latin, Italian, Portuguese Wikisource based on language hint or default to top 5
  5. **Internet Archive**: `archive.org/search?query={query}` URL
  6. **Google Books**: `books.google.com/books?q={query}` URL
  7. **HathiTrust**: `catalog.hathitrust.org/Search/Home?lookfor={query}` URL
  8. **Europeana**: `europeana.eu/search?query={query}` URL
  9. **Library of Congress**: `loc.gov/search/?q={query}` URL
- Archive registry design: Store archives as a list of dicts `ARCHIVE_REGISTRY = [{'name': str, 'url_template': str, 'category': str, 'languages': List[str]}]` so new country-specific archives can be added by appending to the list.
- Return `{'query': str, 'language': str, 'archives': List[Dict], 'priority_note': str}` where each archive entry has `{'name': str, 'category': str, 'search_url': str, 'description': str}`.
- Priority note text: "Academic/critical editions listed first. These are preferred for scholarly apparatus. Free archive versions supplement but don't replace critical editions."
- Return `{'error': msg}` on failure.

**`structure_assessor.py`** — `StructureAssessor` class (DISC-02):
- `assess(document_name: str, description: Optional[str] = None) -> Dict[str, Any]`: Main entry. Accepts document name + optional location/publication info. NOT necessarily a URL — the user describes the document.
- Document type classification (Claude's discretion): Classify as one of: `legal_code` (statutes, decrees — articles/clauses), `treaty` (bilateral/multilateral — articles/provisions), `book` (monograph/chronicle — chapters/sections), `letter` (diplomatic correspondence — short form), `decree` (executive orders — sections/clauses), `other`.
- For each type, return expected structure template:
  - `legal_code`: preamble + numbered articles with clauses
  - `treaty`: preamble + numbered articles + annexes
  - `book`: chapters with sections
  - `letter`: salutation + body paragraphs + closing
  - `decree`: preamble + numbered articles/sections
- `estimate_video_length(section_count: int, document_type: str, scope: str = 'full') -> Dict[str, Any]`: Estimate video length.
  - Formula: Use hybrid approach — base time per section (legal clause ~2 min for read+translate+explain, book chapter ~5-8 min for summary+key passages+analysis) + overhead for intro (2 min) and conclusion (2 min).
  - `scope` parameter: 'full' (entire document) or 'excerpt' (user picks sections later — just show per-section estimates).
  - Return `{'estimated_minutes': int, 'breakdown': Dict, 'scope': str, 'note': str}`.
  - Note: "As long as needed — optimize for completeness, not brevity" (channel philosophy).
- `generate_outline(document_name: str, document_type: str, section_count: int, section_names: Optional[List[str]] = None) -> str`: Generate markdown outline with one-line placeholder summaries per section.
  - For structured docs (legal/treaty/decree): Number each article/section.
  - For books: Chapter headings.
  - Include "[Summary TBD — fill after reading original]" placeholders.
- Return `{'document_name': str, 'document_type': str, 'section_count': int, 'outline': str, 'length_estimate': Dict, 'scope_options': str}` where scope_options explains user can choose full-document or excerpt-based after seeing structure.
- Return `{'error': msg}` on failure.

**`cli.py`** — Unified CLI entry point:
- Use argparse with subcommands: `gap`, `structure`, `archive`.
- `python tools/document_discovery/cli.py gap "Statut des Juifs 1940"` — runs GapChecker.check_gap()

- All output as formatted markdown to stdout (consistent with existing tools like keywords.py).
- `--json` flag on all subcommands for machine-readable output.
- Exit code 0 on success, 1 on error (consistent with keywords.py pattern).
- Handle book-length documents: `python tools/document_discovery/cli.py structure "Brevisima relacion de la destruccion de las Indias" --type book --sections 16` should work for chapters.
