# Plan 04-01: Script File Standardization

**Phase:** 04 - Script Management
**Requirement:** SCRP-01 (One canonical script file per video)

## Objective

Migrate all existing video projects to use `SCRIPT.md` as the single canonical script file, with git tracking history instead of version sprawl (V2, V3, FINAL, FINAL-FINAL).

## Tasks

<task id="1">
**Audit existing script files**

Scan all projects in `video-projects/_IN_PRODUCTION/` for script-related files:
- `02-SCRIPT-DRAFT.md`
- `FINAL-SCRIPT.md`
- Files containing V2, V3, FINAL in name
- Any `.md` files with "script" in the name

Create inventory showing which projects need migration.
</task>

<task id="2">
**Migrate projects to SCRIPT.md pattern**

For each project needing migration:
1. Identify most recent/authoritative script file (by date or content)
2. Rename to `SCRIPT.md` in project root
3. Move old versions to `_archive/` subfolder within project
4. Commit each project migration separately

Skip projects that already have `SCRIPT.md` as canonical (like project 30).
</task>

<task id="3">
**Update templates and commands**

Update any templates or commands that reference old naming:
- Check `/new-video` command for script file references
- Check `/script` command output location
- Update any templates that create `02-SCRIPT-DRAFT.md`

Change references to use `SCRIPT.md`.
</task>

<task id="4">
**Document convention in CLAUDE.md**

Add brief note to CLAUDE.md File Organization section:
- `SCRIPT.md` is canonical script file per project
- Git tracks version history (no V2/V3/FINAL files)
- Old versions archived in `_archive/` subfolder
</task>

## Verification

- [ ] All active projects have single `SCRIPT.md` file
- [ ] No `V2`, `V3`, `FINAL-FINAL` script files in project roots
- [ ] Old versions preserved in `_archive/` folders
- [ ] Templates/commands reference `SCRIPT.md`
- [ ] Convention documented

## must_haves

1. Every project in `_IN_PRODUCTION/` has exactly one `SCRIPT.md` (or no script if pre-scripting phase)
2. No script version sprawl in project roots
3. Commands create `SCRIPT.md`, not numbered variants
