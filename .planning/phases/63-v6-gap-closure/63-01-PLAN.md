Add this public method to the KeywordDB class (near the existing `get_all_video_performance` method around line 1957):

```python
def search_video_performance_by_title(self, title_prefix: str) -> str | None:
    """Look up video_id by title prefix using case-insensitive LIKE match."""
    if not title_prefix:
        return None
    try:
        self._ensure_performance_table()
        prefix = title_prefix[:40]
        cursor = self._conn.cursor()
        cursor.execute(
            "SELECT video_id FROM video_performance "
            "WHERE title LIKE ? COLLATE NOCASE LIMIT 1",
            (f"%{prefix}%",),
        )
        row = cursor.fetchone()
        return row[0] if row else None
    except Exception as exc:
        logger.warning("DB lookup error for '%s': %s", title_prefix, exc)
        return None
```

Note: Do NOT escape single quotes in the prefix — the parameterized query handles that. The old code in ctr_ingest.py had `.replace("'", "''")` which was unnecessary with `?` placeholders.

def _lookup_video_id(db, title: str) -> str | None:

Keep the function signature identical so existing callers and tests are unaffected.
