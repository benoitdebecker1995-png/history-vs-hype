**algo_synthesizer.py** — LLM synthesis of algorithm knowledge:
- `SYNTHESIS_PROMPT` constant from RESEARCH.md: instructs LLM to ONLY use scraped content, ignore training data, note publication dates, flag conflicting claims, output structured JSON
- JSON output schema: `signal_weights`, `pipeline_mechanics` (browse/search/suggested), `longform_specific`, `satisfaction_signals`, `avd_thresholds`, `ctr_thresholds`, `small_channel_notes`, `sources_used`, `confidence`, `synthesis_date`
- `synthesize_algorithm_knowledge(scraped_results: list[dict]) -> dict`:
  - Combine scraped content from all successful results into single text block
  - Skip results with 'error' key
  - If no successful results, return `{'error': 'No algorithm sources available for synthesis'}`
  - Build prompt with scraped text injected
  - Call Claude Code LLM using the slash command pattern (this function prepares the prompt and data structure; the actual LLM call happens in refresh.py which runs in Claude Code context)
  - Actually: since this runs as Python, NOT in Claude Code agent context, the synthesizer should format the prompt and return it for the refresh orchestrator to handle
  - Alternative approach: Use a FALLBACK mode — if no LLM available, extract key patterns using simple text analysis (keyword counting, sentence extraction for terms like "CTR", "AVD", "satisfaction", "browse", "suggested")
  - Implement `synthesize_with_text_analysis(scraped_results: list[dict]) -> dict`:
    - Search for key algorithm terms in scraped text
    - Extract sentences containing algorithm signals
    - Structure into the same JSON schema with confidence='low'
    - This is the primary mode — LLM synthesis is handled by the /intel command when run in Claude Code
  - Return `{'algorithm_model': dict, 'signal_weights': dict, 'longform_insights': list, 'confidence': str, 'sources_used': list}`
- Store `SYNTHESIS_PROMPT` as a constant so it can be used by the /intel command for LLM-powered refresh

**pattern_analyzer.py** — Outlier detection + niche pattern extraction:
- `detect_outliers(videos: list[dict], multiplier: float = 3.0) -> list[dict]`:
  - Group videos by channel_id
  - For each channel: calculate median view count from videos with views > 0
  - Flag videos with views >= median * multiplier as outliers
  - Add `is_outlier: bool` and `outlier_ratio: float` (views / median) to each video
  - Need minimum 3 videos per channel for meaningful median
  - Return modified video list with outlier flags
- `extract_niche_patterns(videos: list[dict]) -> dict`:
  - Calculate format patterns: average duration, median duration, duration distribution buckets (0-10min, 10-20min, 20-30min, 30-45min, 45+min)
  - Extract title patterns: count title formulas (question, how/why, list, colon-split, quote), average title length
  - Detect common topics by keyword extraction from titles (war, empire, roman, medieval, colonial, etc.)
  - Calculate posting frequency per channel (days between uploads)
  - Return `{'format_patterns': dict, 'hook_patterns': dict, 'trending_topics': list}`
- `generate_outlier_analysis(outlier_videos: list[dict]) -> list[dict]`:
  - For each outlier, note: title, channel, views, outlier_ratio, duration_seconds
  - Add `possible_reasons: list[str]` based on heuristics:
    - Title contains question → "Question-format title"
    - Duration > 30min → "Long-form deep dive"
    - Title contains numbers/dates → "Specific factual claim"
    - Title contains "vs" or "versus" → "Comparison format"
  - Return list of outlier analysis dicts

**kb_exporter.py** — Generate agent-readable Markdown from SQLite:
- `export_kb_to_markdown(db_path: str = None, output_path: str = None) -> dict`:
  - Default db_path: `tools/intel/intel.db`
  - Default output_path: `channel-data/youtube-intelligence.md`
  - Use KBStore to read latest data from all tables
  - Generate Markdown with these sections:
    1. **Header**: "YouTube Intelligence KB — Last refreshed: {date}" + "This file is auto-generated. Do not edit manually."
    2. **Algorithm Mechanics**: Signal weights table, pipeline mechanics (browse/search/suggested), longform-specific insights, satisfaction signals, AVD/CTR thresholds
    3. **Competitor Landscape**: Table of tracked channels with last upload date, avg views, outlier count. Then recent outlier videos table (title, channel, views, outlier_ratio, duration)
    4. **Niche Patterns**: Format stats (avg/median duration, duration distribution), title formula distribution, trending topics by frequency, posting cadence
    5. **Outlier Analysis**: For each outlier, brief analysis of why it may have performed
  - Target under 3000 words total (per RESEARCH.md Pitfall 5) — use concise tables and bullets, not prose
  - Use scannable format: tables for data, bullets for insights
  - Return `{'written_to': path, 'sections': list, 'word_count': int}`
- Helper functions: `_render_algo_section(snapshot)`, `_render_competitor_section(channels, videos)`, `_render_niche_section(snapshot)`, `_render_outlier_section(outliers)`

**refresh.py** — Full refresh orchestrator:
- `run_refresh(force: bool = False) -> dict`:
  - Check staleness via KBStore.is_stale() unless force=True
  - If not stale and not force: return `{'skipped': True, 'reason': 'KB is current', 'last_refresh': timestamp}`
  - Phase 1: Scrape algorithm sources (algo_scraper.scrape_all_sources)
  - Phase 2: Synthesize algorithm knowledge (algo_synthesizer.synthesize_with_text_analysis)
  - Phase 3: Save algorithm snapshot to KBStore
  - Phase 4: Fetch all competitors (competitor_tracker.fetch_all_competitors)
  - Phase 5: Purge old competitor videos (KBStore.purge_competitor_videos) — per user decision "purge outdated data"
  - Phase 6: Save new competitor videos to KBStore
  - Phase 7: Run outlier detection (pattern_analyzer.detect_outliers) and update is_outlier flags
  - Phase 8: Extract niche patterns (pattern_analyzer.extract_niche_patterns) and save snapshot
  - Phase 9: Export to Markdown (kb_exporter.export_kb_to_markdown)
  - Phase 10: Update last_refresh timestamp
  - Return change summary: `{'refreshed': True, 'algo_sources_scraped': int, 'channels_fetched': int, 'videos_total': int, 'outliers_found': int, 'kb_exported_to': path, 'errors': list}`
- `get_refresh_summary(store: KBStore) -> dict`:
  - Compare current state to what was stored: new videos count, new outliers, algorithm changes
  - Return summary dict for display after refresh
- `ensure_channels_loaded(store: KBStore)`:
  - Check if competitor_channels table has entries
  - If empty, load from competitor_channels.json and insert via save_competitor_channel
  - Called at start of refresh to bootstrap channel config
- Error handling: catch all exceptions per module, collect errors list, continue with remaining modules
- Print progress to stdout: "Scraping algorithm sources...", "Fetching competitor feeds...", etc.
