- `fetch_channel_rss(channel_id: str, channel_name: str) -> dict`:
  - Parse RSS feed with feedparser
  - Extract: video_id (entry.yt_videoid), title, published date, description (first 500 chars)
  - Return `{'channel_id': str, 'channel_name': str, 'videos': list[dict]}` or `{'error': msg}`
  - Handle feedparser.bozo flag for malformed feeds
- `enrich_videos_with_metadata(video_ids: list[str]) -> dict`:
  - Import `get_authenticated_service` from `tools/youtube-analytics/auth.py` using sys.path manipulation (matching existing pattern in analyze.py)
  - Call `youtube.videos().list(part='statistics,contentDetails', id=','.join(video_ids[:50]))` — 1 quota unit per call
  - Parse ISO 8601 duration (PT15M30S) to seconds using regex
  - Return `{video_id: {'views': int, 'likes': int, 'duration_seconds': int}}` or `{'error': msg}`
  - Batch in groups of 50 if more than 50 IDs
  - Graceful fallback if YouTube API auth fails: return `{'error': 'YouTube API not authenticated'}` and proceed with RSS-only data

  - Return `{'channels_fetched': int, 'videos_total': int, 'videos': list[dict], 'errors': list[str]}`
