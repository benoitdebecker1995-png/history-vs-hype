Create `tools/dashboard/project_scanner.py` with the following components (all stdlib — pathlib, time, re, datetime):

**Constants:**
- `PHASE_PRIORITY` dict mapping phase names to priority numbers: filming-ready=1, fact-checked=2, scripting=3, research=4, idea=5
- `NEXT_ACTION` dict mapping phase names to suggested commands: filming-ready="Film it", fact-checked="/prep --edit-guide", scripting="/verify", research="/script", idea="/research --new"
- `STALE_THRESHOLD_DAYS = 14`

**Functions:**

1. `detect_phase(files: set[str]) -> str` — Detect project phase from file names present:
   - If `FINAL-SCRIPT.md` in files OR any file contains "TELEPROMPTER" (case-insensitive) → "filming-ready"
   - If `03-FACT-CHECK-VERIFICATION.md` in files AND (`02-SCRIPT-DRAFT.md` in files OR `SCRIPT.md` in files) → "fact-checked"
   - If `02-SCRIPT-DRAFT.md` in files OR `SCRIPT.md` in files → "scripting"
   - If `01-VERIFIED-RESEARCH.md` in files → "research"
   - Else → "idea"

2. `days_since_activity(folder: Path) -> int` — Calculate days since most recent top-level file modification:
   - Iterate only top-level files (not subdirectories, not recursive)
   - Use `time.time() - stat().st_mtime` for cross-platform correctness (Pitfall 1 from RESEARCH)
   - Return `int(delta / 86400)`
   - Return 999 sentinel if no files found or OSError

3. `extract_topic_slug(folder_name: str) -> str` — Extract topic words from folder name:
   - Strip leading number prefix: `re.sub(r'^\d+-', '', folder_name)`
   - Strip trailing year: `re.sub(r'-\d{4}$', '', slug)`
   - Replace hyphens with spaces
   - Return lowercased topic words string

4. `extract_intel_topics(intel_path: Path) -> list[str]` — Read youtube-intelligence.md and extract trending topic words:
   - Return empty list if file missing (never raise)
   - Scan line-by-line for "Trending Topics" header
   - Parse table rows: split by `|`, extract first column (skip header row and separator)
   - Stop at next `#` heading
   - Wrap entire read in try/except returning empty list on any error

5. `scan_projects(project_root: str = '.') -> dict` — Main scanner:
   - Scan `video-projects/_IN_PRODUCTION/` for directories (skip non-dirs, skip items starting with "README")
   - For each folder: detect phase, calculate days_since, check if published (POST-PUBLISH-ANALYSIS.md present), extract topic slug
   - Build project dict with keys: name, phase, priority, next_action, days_since, stale (bool), published (bool), topic_slug
   - Sort: published projects last, then by priority ascending, then by days_since ascending (most recent first)
   - Also scan `_READY_TO_FILM/` for any folders there
   - Extract intel topics from `channel-data/youtube-intelligence.md`
   - Return dict with keys: `projects` (list), `ready_to_film` (list of names), `intel_topics` (list of strings)

6. `format_dashboard(data: dict) -> str` — Format scan results as Markdown dashboard:
   - Header: `## Project Dashboard`
   - Split projects into active (not published) and published lists
   - Active projects table with columns: Project, Phase, Next Action, Activity
   - Activity column shows days count, with "STALE" suffix if >14 days
   - Intel cross-reference: for each active project, check if any intel topic word appears in the project's topic_slug. If match found, append a brief note after the table: "Trending topic match: [topic] — [project names]"
   - Published projects shown as footer summary line: "N published (not archived): name1, name2, ..."
   - Idea-phase projects: show count at bottom if >5 ideas exist: "N idea-phase projects not shown" — only suppress idea projects from the main table if there are more than 10 active non-idea projects (keep the table scannable). Otherwise show all.
   - _READY_TO_FILM footer if any exist
   - Return complete Markdown string

**Anti-patterns to avoid (from RESEARCH.md):**
- Do NOT recursively walk subdirectories for mtime (Pitfall 3)
- Do NOT mix timezone-aware and naive datetimes (Pitfall 1) — use `time.time()` consistently
- Do NOT filter by file extension — just check top-level file names
- Do NOT read file contents except for intel topics — speed matters with 39 projects
