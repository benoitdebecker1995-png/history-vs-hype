**Functions:**

`check_metadata_consistency(metadata: dict) -> dict`:
Input metadata dict:
```python
{
    'title': 'The Dark Ages Myth: What Medieval Europe Was Really Like',
    'description': 'Long description text...',
    'tags': ['dark ages', 'medieval history', 'myth busting', ...],
    'primary_keyword': 'dark ages myth'  # Optional, will infer from title if not provided
}
```

Checks to perform:
1. **Primary keyword in title** (HIGH severity if missing)
   - Should be in first half of title preferably
2. **Primary keyword in description first 200 chars** (HIGH severity)
3. **Primary keyword in tags** (MEDIUM severity)
4. **Keyword stuffing check** (HIGH if density > 2%)
   - Count keyword occurrences / total word count
5. **Title-tag overlap** (MEDIUM if < 3 words overlap)
   - Extract significant words from title, check tag coverage
6. **Description length** (WARNING if < 200 words)
7. **Tag count** (WARNING if < 5 or > 30)

Return:
```python
{
    'passed': True/False,  # False if any HIGH severity issues
    'issues': [
        {
            'severity': 'HIGH' | 'MEDIUM' | 'WARNING',
            'element': 'Title' | 'Description' | 'Tags',
            'issue': 'Primary keyword "X" not found in title',
            'fix': 'Add primary keyword to first half of title'
        }
    ],
    'stats': {
        'title_length': 65,
        'description_words': 250,
        'tag_count': 15,
        'keyword_density': 1.2,
        'title_tag_overlap': 5
    },
    'summary': '3 issues found (1 high severity)'
}
```

`infer_primary_keyword(title: str) -> str`:
- Extract likely primary keyword from title
- Use longest phrase that appears to be the topic
- Fall back to title cleaning (remove common words)

`format_metadata_report(result: dict, format: str = 'markdown') -> str`:
- Format check results as markdown or JSON
- Use table for issues list
- Show pass/fail verdict prominently

**CLI interface:**
```

python metadata_checker.py --file YOUTUBE-METADATA.md
python metadata_checker.py --file YOUTUBE-METADATA.md --json
```

When reading from file, parse markdown format:

- Extract description from body

```markdown
# /discover - Keyword Research & Topic Discovery

Research keywords and evaluate topics for channel fit.

## Usage

```
/discover TOPIC           # Full keyword research workflow
/discover --autocomplete "seed phrase"  # Extract autocomplete suggestions
/discover --intent "title or query"     # Classify search intent
/discover --check YOUTUBE-METADATA.md   # Pre-publish metadata check
/discover --vidiq "topic"               # Start VidIQ guided workflow
```

## Flags

| Flag | Description |
|------|-------------|
| `--autocomplete` | Extract YouTube autocomplete suggestions for seed |
| `--intent` | Classify search intent and channel DNA fit |
| `--check` | Run metadata consistency check |
| `--vidiq` | Generate VidIQ guided workflow prompts |
| `--save` | Save results to database |
| `--json` | Output as JSON instead of markdown |
| `-q` | Quick mode (summary only) |

## Examples

### Full Research Workflow

```
/discover "dark ages medieval myth"
```

Runs:
1. Autocomplete extraction for keyword variants
2. Intent classification for each variant
3. Channel DNA fit scoring
4. Saves keywords to database

### Pre-publish Check

```
/discover --check video-projects/_IN_PRODUCTION/19-flat-earth-medieval-2025/YOUTUBE-METADATA.md
```

Validates:
- Primary keyword in title, description, tags
- No keyword stuffing (< 2% density)
- Title-tag word overlap
- Description length adequate

### VidIQ Workflow

```
/discover --vidiq "dark ages myth"
```

Generates step-by-step prompts for manual VidIQ data collection.

## Execution

Run the appropriate tool based on flags:

```bash
# Autocomplete
cd tools/discovery && python autocomplete.py "TOPIC" --save

# Intent
cd tools/discovery && python intent_mapper.py "TOPIC"

# Metadata check
cd tools/discovery && python metadata_checker.py --file "PATH"

# VidIQ workflow
cd tools/discovery && python vidiq_workflow.py "TOPIC"

# Full workflow (default)
cd tools/discovery && python keywords.py research "TOPIC"
```

## Output

Results saved to:
- Keywords: `tools/discovery/keywords.db`
- VidIQ data: `video-projects/[project]/vidiq-data.json` or `channel-data/vidiq-research/`
- Metadata reports: stdout (or `--save` to project folder)

## Related Commands

- `/research --new` - Start new video project (includes discovery)
- `/analyze` - Post-publish analysis with discovery diagnostics
- `/publish` - Pre-publish checklist (includes metadata check)
```
