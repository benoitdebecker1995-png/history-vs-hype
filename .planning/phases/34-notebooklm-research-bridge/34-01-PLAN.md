  <name>Task 1: Create notebooklm_bridge.py with Claude API source generation</name>
  <files>tools/notebooklm_bridge.py</files>
  <action>
Create `tools/notebooklm_bridge.py` (~250 LOC) implementing academic source list generation via Claude API.

**Module structure:**
1. `generate_source_list(topic, video_type, num_sources)` — Core function. Calls Claude API with a structured prompt that requests university press sources organized by tier. Returns error dict pattern: `{'status': 'success', 'content': str, 'model': str}` or `{'error': msg}`. Use `anthropic.Anthropic()` client (reads ANTHROPIC_API_KEY from env automatically). Model: use `claude-sonnet-4-20250514` (cost-efficient for list generation — source generation does not need Opus-level reasoning). Max tokens: 4000.

2. `write_source_list(content, output_dir, topic)` — Writes the source list to `NOTEBOOKLM-SOURCE-LIST.md` in the specified output directory. Adds header with generation metadata (date, topic, video type). Returns `{'status': 'success', 'path': str}` or `{'error': msg}`. Use pathlib for all path operations. UTF-8 encoding mandatory.

3. `main()` — argparse CLI entry point with these arguments:
   - `topic` (positional, required): The video topic string
   - `--type` (optional, default="general"): Video type — choices: territorial, ideological, fact-check, general
   - `--output` (optional, default=cwd): Output directory for the source list file
   - `--sources` (optional, default=15): Number of sources to request (10-20 range)
   - `--dry-run` (optional flag): Print to stdout instead of writing file

**Prompt engineering for the Claude API call:**
Build a detailed system message that instructs Claude to:
- Generate academic sources from university presses ONLY (Cambridge, Oxford, Chicago, Harvard, Yale, Princeton, Stanford, Cornell)
- Organize by tier matching NOTEBOOKLM-SOURCE-STANDARDS.md hierarchy:
  - Tier 1: Primary Sources (treaties, court rulings, official documents, contemporary accounts)
  - Tier 2: Academic Monographs (peer-reviewed, university press)
  - Tier 3: Supplementary (think tank reports, legal encyclopedias, expert interviews)
- For each source include: title, author with credentials, publisher, year, ISBN (if available), estimated price, purchase/download link (Amazon or publisher direct), relevance explanation
- Use markdown formatting matching the `[P1]`, `[A1]` naming convention from NOTEBOOKLM-SOURCE-STANDARDS.md
- Include a SOURCE QUALITY CHECK table at the top (same format as in `/sources` command)
- Tailor recommendations based on video_type:
  - territorial: emphasize treaties, boundary surveys, ICJ cases, colonial archives
  - ideological: emphasize historiography, manuscript evidence, scholarly debates
  - fact-check: emphasize primary source documents, court filings, official records
  - general: balanced mix

**Error handling:**
- Missing ANTHROPIC_API_KEY: return `{'error': 'ANTHROPIC_API_KEY not set. Export your API key: export ANTHROPIC_API_KEY=sk-...'}`
- API errors (rate limit, auth, network): catch `anthropic.APIError` and return error dict
- File write errors: catch OSError and return error dict
- All exceptions: never raise, always return error dict

**CLI output:**
- On success: print source count, output file path, and a "Next steps" message pointing to NotebookLM upload
- On error: print error message to stderr and exit with code 1
- On dry-run: print source list to stdout

**Do NOT:**
- Use web search or scraping (per user decision: Claude API generation only)
- Import any external libraries beyond `anthropic` and stdlib
- Access clipboard
- Write to VERIFIED-RESEARCH.md (separate extraction file only)
