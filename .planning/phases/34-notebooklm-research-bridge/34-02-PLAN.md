  <action>
Create `tools/citation_extractor.py` (~150 LOC) that parses NotebookLM chat output and extracts citations into VERIFIED-RESEARCH.md-compatible format.

**This replaces the Phase 14 PowerShell script** (`.claude/tools/extract-citations.ps1`) with a cross-platform Python tool that produces output in the correct VERIFIED-RESEARCH.md format.

**Module structure:**

1. `parse_citations(content: str) -> List[Dict]` — Core parsing function. Extracts citation markers and source references from NotebookLM output text.

   **Parsing strategy (multi-pattern, handle NotebookLM format variations):**
   - Pattern A: Source legend at bottom — `[1] Author, Book Title, p. 45` or `[1] Author, Book Title, page 45` or `[1] Author, Book Title, pp. 45-46`
   - Pattern B: SOURCES section with numbered list — `1. Author, "Book Title" (Year), p. 45`
   - Pattern C: Inline citation with parenthetical — `(Author, Year, p. 45)`

   Try Pattern A first (most common NotebookLM format), fall back to Pattern B (matches existing PowerShell script's SOURCES: section parsing), then Pattern C.

   For claims: extract sentences containing `[N]` markers. Clean the claim text by removing the citation marker itself.

   Return list of dicts: `[{'claim': str, 'source': str, 'page': str, 'citation_num': str, 'status': 'NEEDS REVIEW'}]`

2. `extract_citations(input_path: str) -> Dict[str, Any]` — File I/O wrapper. Reads input file (UTF-8, with fallback to latin-1 if UnicodeDecodeError), calls parse_citations(), returns error dict on failure. Returns `{'status': 'success', 'citations': list, 'count': int, 'warnings': list}` or `{'error': msg}`.

   Warnings for: citations in text with no matching source in legend, sources in legend not referenced in text, encoding fallback used.

3. `write_extractions(citations: list, output_path: Path, input_filename: str)` — Writes NOTEBOOKLM-EXTRACTIONS.md in VERIFIED-RESEARCH.md-compatible format.

   **Output format per extracted claim:**
   ```markdown
   ### Claim N
   **Claim:** [claim text]
   **Source:** [author, title], p. [page]
   **Status:** NEEDS REVIEW
   **Verification:**
   - [ ] Verify claim accuracy against source
   - [ ] Confirm page number
   - [ ] Update status: VERIFIED / UNVERIFIABLE / PARTIALLY TRUE
   - [ ] Copy to 01-VERIFIED-RESEARCH.md when verified
   ```

   Header includes: extraction date, source file, total claims extracted, warnings.

4. `main()` — argparse CLI entry point:
   - `input` (positional, required): Path to file with pasted NotebookLM output
   - `--output` (optional): Output file path (default: NOTEBOOKLM-EXTRACTIONS.md in same directory as input)
   - `--format` (optional, choices: "detailed", "compact"): Output format. "detailed" = full verification checklist per claim (default). "compact" = table format for quick review.
   - `--stats-only` (optional flag): Print extraction statistics without writing file

**Error handling:**
- File not found: return `{'error': f'File not found: {path}'}`
- Empty file: return `{'error': 'Input file is empty'}`
- No citations found: return `{'error': 'No citations found. Check that the file contains NotebookLM output with [1], [2] citation markers or a SOURCES: section.'}`
- Encoding errors: try UTF-8 first, fall back to latin-1, warn if fallback used
- All errors: error dict pattern, never raise

**CLI output:**
- On success: print extraction count, output path, and warnings if any
- On error: print error to stderr, exit code 1
- On --stats-only: print citation count, source count, and any warnings

**Do NOT:**
- Use NLP libraries (spaCy, NLTK) for citation parsing — simple regex is sufficient per research findings
- Write directly to VERIFIED-RESEARCH.md (user decision: separate extraction file for review)
- Access clipboard
- Import any external libraries beyond stdlib
