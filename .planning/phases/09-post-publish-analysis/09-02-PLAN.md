<task type="auto">
  <name>Task 1: Create analyze.py - Complete analysis orchestrator</name>
  <files>tools/youtube-analytics/analyze.py</files>
  <action>
Create the main orchestrator script that combines all data sources into a complete post-publish analysis.

**Architecture:**
```
analyze.py orchestrates:
  video_report.py -> engagement, retention, CTR data
  comments.py -> categorized comments
  channel_averages.py -> benchmarks for comparison
  + generates lessons based on data patterns
```

**Function: extract_video_id(url_or_id)**
- Parse YouTube URLs to extract video ID (supports youtube.com/watch, youtu.be, shorts, embed)
- Return video ID unchanged if already 11-character ID
- Use urllib.parse + regex (see 09-RESEARCH.md Pattern 1)

**Function: run_analysis(video_id_or_url)**
Main entry point. Steps:
1. Extract video ID from URL if needed
2. Call generate_video_report() from video_report.py (engagement, retention, CTR)
3. Call fetch_and_categorize_comments() from comments.py
4. Call get_channel_averages() from channel_averages.py
5. Call compare_to_channel() to get benchmark comparisons
6. Generate lessons using generate_lessons()
7. Combine into complete analysis dict

Return structure:
```python
{
    'video_id': '...',
    'title': '...',
    'fetched_at': 'ISO timestamp',

    # From video_report
    'engagement': {...},
    'ctr': {...},
    'retention': {...},

    # From channel_averages
    'benchmarks': {
        'channel_averages': {...},
        'comparison': {...}  # above/below/at_average for each metric
    },

    # From comments
    'comments': {
        'total': N,
        'questions': [...],
        'objections': [...],
        'requests': [...]
    },

    # Generated
    'lessons': {
        'observations': [...],  # What the data shows
        'actionable': [...]     # What to do next time
    },

    'errors': [...]
}
```

**Function: generate_lessons(analysis_data)**
Generate automated lessons based on analysis patterns. Rules:

Retention observations:
- If avg_retention >= 35%: "Strong retention ({X}%) - hook and pacing working well"
- If avg_retention < 25%: "Retention needs work ({X}%) - review first 30 seconds"
- If biggest drop > 10%: "Major drop-off at {position}% ({location}) - review that section"
- If drop count > 5: "Multiple drop-off points ({N}) - consider tightening overall pacing"

Engagement observations:
- If subs_gained > views/100: "Strong subscriber conversion ({X} per 100 views)"
- If engagement_rate > 5%: "Excellent engagement ({X}%)"
- If engagement_rate < 1%: "Low engagement ({X}%) - consider adding CTA"

Benchmark observations:
- If views above average by >50%: "Outperforming channel average by {X}%"
- If views below average by >50%: "Underperforming vs channel average"

Comment observations:
- If questions > 5: "{N} questions detected - consider follow-up content"
- If objections > 3: "{N} objections raised - review accuracy"
- If requests > 3: "{N} content requests - note for future topics"

Actionable takeaways (derive from observations):
- Based on retention: "Consider [shorter intros / tighter pacing / more pattern interrupts]"
- Based on comments: "Address top {N} questions in pinned comment or follow-up"
- Based on performance: "This topic type [works well / needs refinement]"

**Function: format_analysis_markdown(analysis)**
Format complete analysis as Markdown for human reading. Include:
- Quick summary (above/below benchmarks at a glance)
- Performance table with benchmark comparisons (value | channel avg | vs avg)
- Retention section with ASCII curve (use simple custom implementation, no external deps)
- All drop-off points with timestamps and video position
- Full comment lists under each category (not just counts)
- Lessons section with observations and actionable items

**ASCII retention curve:**
Use custom implementation from 09-RESEARCH.md Pattern 5 (Option B - no dependencies).
Show curve with 10 rows, 60 columns, percentage labels.

**CLI interface:**
- `python analyze.py VIDEO_ID` - JSON output
- `python analyze.py VIDEO_ID --markdown` - Markdown output
- `python analyze.py https://youtu.be/VIDEO_ID` - Accept URLs
- Include help message if no args

**Error handling:**
- Partial failures don't crash (individual errors captured in 'errors' array)
- If video_report fails completely, still try comments and benchmarks
- Always return a valid dict structure
