<task type="auto">
  <name>Task 1: Create comments.py - Comment fetcher and categorizer</name>
  <files>tools/youtube-analytics/comments.py</files>
  <action>
Create a Python script that fetches and categorizes YouTube video comments.

**Requirements:**
1. Use YouTube Data API v3 `commentThreads.list()` endpoint (NOT Analytics API)
2. Import `get_authenticated_service` from `auth.py` (already supports 'youtube', 'v3')
3. Follow error handling pattern from metrics.py (return dict with 'error' key, not exceptions)

**Function: fetch_video_comments(video_id, max_comments=100)**
- Fetch up to max_comments using pagination via `nextPageToken`
- Use `order='relevance'` to get top comments by likes/replies first
- Return list of dicts: `{text, author, likes, published_at, reply_count}`
- Handle errors gracefully (return `{'error': 'message', 'video_id': video_id}`)

**Function: categorize_comments(comments)**
- Classify into three categories: questions, objections, requests
- Question patterns: contains `?`, starts with question words (what/why/how/when/where/who/can/could/is/are/do)
- Objection patterns: contains disagreement words (wrong, incorrect, false, disagree, actually, mistake, error)
- Request patterns: contains request words (please, could you, video on, video about, cover, suggestion)
- Return dict: `{'questions': [...], 'objections': [...], 'requests': [...], 'other': [...]}`

**Function: fetch_and_categorize_comments(video_id, max_comments=100)**
- Main entry point combining both functions
- Return: `{'video_id': ..., 'total_fetched': N, 'categories': {...}, 'category_counts': {...}}`

**CLI interface:**
- `python comments.py VIDEO_ID` - fetch and categorize, output JSON
- Include usage help if no args

**Style requirements:**
- Match Phase 8 script style (docstrings, type hints, same error patterns)
- Use `from auth import get_authenticated_service`
- Don't install new packages (YouTube Data API already works via auth.py)

<task type="auto">
  <name>Task 2: Create channel_averages.py - Benchmark calculator</name>
  <files>tools/youtube-analytics/channel_averages.py</files>
  <action>
Create a Python script that calculates channel-wide benchmark averages from recent videos.

**Requirements:**
1. Use YouTube Data API v3 `search.list()` to get recent video IDs for the channel
2. Use existing `get_video_metrics()` from metrics.py to fetch each video's data
3. Calculate averages across videos for benchmark comparisons

**Function: get_recent_video_ids(max_videos=10)**
- Use `youtube.search().list(part='id', forMine=True, type='video', order='date', maxResults=max_videos)`
- Return list of video IDs (strings)
- Handle auth failure gracefully

**Function: get_channel_averages(last_n_videos=10)**
- Call get_recent_video_ids() to get video list
- For each video, call get_video_metrics() from metrics.py
- Skip videos with errors (don't crash on individual failures)
- Require minimum 3 videos for meaningful averages (return error if fewer)
- Calculate and return:
  ```python
  {
      'sample_size': int,
      'avg_views': float,
      'avg_watch_time_minutes': float,
      'avg_likes': float,
      'avg_comments': float,
      'avg_subscribers_gained': float,
      'avg_retention': float,  # if available
      'video_ids_sampled': [...],
      'fetched_at': 'ISO timestamp'
  }
  ```

**Function: compare_to_channel(video_metrics, channel_averages)**
- Compare a single video's metrics to channel averages
- Return dict with comparison:
  ```python
  {
      'views': {'value': 1500, 'average': 1000, 'vs_average': 'above', 'delta_percent': 50.0},
      'watch_time': {...},
      'likes': {...},
      ...
  }
  ```
- Use 'above', 'below', 'at_average' (within 5% = at_average)

**CLI interface:**
- `python channel_averages.py` - fetch and display channel averages
- `python channel_averages.py --compare VIDEO_ID` - show video vs averages comparison

**Style requirements:**
- Match Phase 8 patterns (error dicts, type hints, docstrings)
- Import from metrics.py and auth.py (no new packages)
