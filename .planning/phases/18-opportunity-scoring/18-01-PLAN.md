   - Check keyword.lower() against CHANNEL_DNA_VIOLATIONS
   - Return True if clickbait keywords present
   - Return True if news_first patterns match
   - Return True if keyword STARTS WITH politician name (not just contains)
   - Return (is_violation: bool, violation_type: str, matched_keyword: str)

4. Implement `score_opportunity(keyword_id, demand_data, competition_data, constraints)`:
   - STEP 1: Hard constraints (per RESEARCH.md Pattern 1)
     - If constraints['is_production_blocked'] True: return score=None, category='Blocked', block_reason='Animation required'
     - If _violates_channel_dna(): return score=None, category='Blocked', block_reason='Channel DNA violation: {type}'
   - STEP 2: Normalize inputs
     - demand_normalized = demand_data.get('search_volume_proxy', 0)  # Already 0-100
     - gap_normalized = competition_data.get('differentiation_score', 0) * 100  # 0-1 to 0-100
     - fit_normalized = constraints.get('document_score', 2) * 25  # 0-4 to 0-100
   - STEP 3: Calculate weighted SAW score
     - opportunity_score = sum(normalized * weight for each component)
   - STEP 4: Categorize per RESEARCH.md thresholds
     - >=70: 'Excellent', >=50: 'Good', >=30: 'Fair', <30: 'Poor'
   - STEP 5: Track max data_age_days across all inputs
   - Return complete result dict per RESEARCH.md code example

5. Implement `save_opportunity_score(keyword_id, score_result)`:
   - Update keywords table with opportunity_score_final and opportunity_category
   - Use existing add_opportunity_score for detailed components
   - Transition lifecycle state to 'ANALYZED' if currently 'DISCOVERED'

6. Add module docstring with usage examples:
