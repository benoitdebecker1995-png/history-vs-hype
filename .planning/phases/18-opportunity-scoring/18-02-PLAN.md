   ```python
   """
   Opportunity Orchestrator Module

   Combines Phase 15-17 modules into a single analysis pipeline:
   - Demand analysis (Phase 15)
   - Competition analysis (Phase 16)
   - Production constraints (Phase 17)
   - Opportunity scoring (Phase 18)

   Usage:
       CLI:
           python orchestrator.py "dark ages myth"
           python orchestrator.py "treaty of versailles" --report
           python orchestrator.py --list-state ANALYZED

       Python:
           from tools.discovery.orchestrator import OpportunityOrchestrator
           from tools.discovery.database import KeywordDB

           db = KeywordDB()
           orch = OpportunityOrchestrator(db)
           result = orch.analyze_opportunity("dark ages myth")
   """
   ```

2. Create OpportunityOrchestrator class:
   ```python
   class OpportunityOrchestrator:
       def __init__(self, db: KeywordDB):
           self.db = db
           self.demand = DemandAnalyzer(db)
           self.competition = CompetitionAnalyzer()
           self.scorer = OpportunityScorer(db)
   ```

3. Implement `analyze_opportunity(keyword, force_refresh=False)`:
   - Step 1: Run demand analysis (Phase 15)
     - `demand_data = self.demand.analyze_keyword(keyword, force_refresh)`
     - Return early if error
   - Step 2: Get keyword_id from database
   - Step 3: Run competition analysis (Phase 16)
     - `comp_data = self.competition.analyze_competition(keyword)`
     - Return early if error
   - Step 4: Get/evaluate production constraints (Phase 17)
     - Check db.get_production_constraints(keyword_id)
     - If None, run evaluate_production_constraints() and store result
   - Step 5: Score opportunity (Phase 18)
     - `score_result = self.scorer.score_opportunity(keyword_id, demand_data, comp_data, constraints)`
   - Step 6: Save score and update lifecycle
     - `self.scorer.save_opportunity_score(keyword_id, score_result)`
   - Step 7: Build complete result dict:
     ```python
     {
         'keyword': keyword,
         'keyword_id': keyword_id,
         'opportunity_score': score_result['opportunity_score'],
         'category': score_result['category'],
         'is_blocked': score_result.get('is_blocked', False),
         'block_reason': score_result.get('block_reason'),
         'components': score_result.get('components', {}),
         'demand': {
             'search_volume_proxy': demand_data['search_volume_proxy'],
             'trend_direction': demand_data['trend_direction'],
             'opportunity_ratio': demand_data['opportunity_ratio']
         },
         'competition': {
             'video_count': comp_data['video_count'],
             'channel_count': comp_data['channel_count'],
             'quality_video_count': comp_data.get('quality_video_count', 0),
             'differentiation_score': comp_data.get('differentiation_score', 0),
             'recommended_angle': comp_data.get('recommended_angle', 'unknown')
         },
         'production': {
             'document_score': constraints.get('document_score', 2),
             'is_animation_blocked': constraints.get('animation_required', False),
             'source_hints': constraints.get('source_hints', {})
         },
         'lifecycle_state': self.db.get_lifecycle_state(keyword_id),
         'data_age_days': score_result.get('data_age_days', 0),
         'warnings': score_result.get('warnings', []),
         'fetched_at': datetime.utcnow().isoformat() + 'Z'
     }
     ```

4. Implement `transition_keyword(keyword, new_state)`:
   - Get keyword_id from db
   - Call db.set_lifecycle_state(keyword_id, new_state)
   - Return result

5. Implement `list_by_state(state, limit=50)`:
   - Call db.get_keywords_by_lifecycle(state, limit)
   - Return list of keywords with opportunity scores

6. Implement `generate_report(keyword, output_path=None)`:
   - Run analyze_opportunity(keyword)
   - Load Jinja2 template from templates/opportunity_report.md.j2
   - Render template with analysis data
   - Write to output_path if provided, else return string

7. Add main() CLI entry point with argparse:
   - `python orchestrator.py "keyword"` -- Basic analysis with table output
   - `python orchestrator.py "keyword" --report` -- Generate Markdown report
   - `python orchestrator.py "keyword" --report --output FILE` -- Save report to file
   - `python orchestrator.py "keyword" --refresh` -- Force refresh all cached data
   - `python orchestrator.py --list-state STATE` -- List keywords by lifecycle state
   - `python orchestrator.py --transition KEYWORD STATE` -- Transition keyword to new state
   - `python orchestrator.py "keyword" --json` -- JSON output

Use ASCII-safe output for Windows compatibility (avoid Unicode arrows/bars in CLI, use # and - for progress bars).

```jinja2
{% for name, comp in data.components.items() %}

| Metric | Value |

| Metric | Value |

{% if data.production.source_hints and data.production.source_hints.queries %}
### Source Hints

Try these searches to verify academic source availability:

{% for query in data.production.source_hints.queries[:3] %}

---

{% if data.data_age_days > 7 %}
**Warning:** Data is {{ data.data_age_days }} days old. Consider running with `--refresh`.
{% else %}
Data age: {{ data.data_age_days }} day(s) - within freshness threshold.
{% endif %}

{% if data.warnings %}
### Warnings

{% for warning in data.warnings %}
- {{ warning }}
{% endfor %}
{% endif %}

{% if data.is_blocked %}
1. This topic requires a different format (animation). Consider reframing or choosing a different topic.
2. Search for document-heavy alternatives in the same niche.
{% elif data.category == 'Excellent' %}
1. **Proceed to research phase** - High opportunity score indicates strong potential.
2. Run `/research --new` with this topic.
3. Verify source availability with the search queries above.
{% elif data.category == 'Good' %}
1. Consider proceeding if topic aligns with content calendar.
2. Verify source availability before committing research hours.
3. Check if reframing could improve document score.
{% elif data.category == 'Fair' %}
1. Review competition more carefully - may need stronger differentiation.
2. Consider whether a related topic scores higher.
3. Verify source availability is adequate.
{% else %}
1. Low opportunity - consider alternative topics.
2. High competition or low demand makes this a risky investment.
{% endif %}
```
