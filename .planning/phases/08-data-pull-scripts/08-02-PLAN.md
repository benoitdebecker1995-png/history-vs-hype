Create `tools/youtube-analytics/retention.py` with:


2. **Main function `get_retention_data(video_id, start_date=None, end_date=None)`:**
   - Default date range: 2020-01-01 to today (full video lifetime)
   - Query YouTube Analytics API with (per research doc):
     - `ids='channel==MINE'`
     - `metrics='audienceWatchRatio,relativeRetentionPerformance'`
     - `dimensions='elapsedVideoTimeRatio'`
     - `filters=f'video=={video_id};audienceType==ORGANIC'`
     - `maxResults=200` (API max for retention)
     - `sort='elapsedVideoTimeRatio'` (REQUIRED for retention reports)

3. **Return structure:**
   ```python
   {
       "video_id": "...",
       "data_points": [
           {
               "position": 0.0,  # 0% of video
               "retention": 1.0,  # 100% of viewers
               "relative": 0.95  # vs similar videos
           },
           {
               "position": 0.05,  # 5% of video
               "retention": 0.85,
               "relative": 0.92
           },
           # ... ~100 data points
       ],
       "summary": {
           "avg_retention": float,  # mean of all retention values
           "min_retention": float,
           "final_retention": float  # retention at video end
       },
       "date_range": {"start": "YYYY-MM-DD", "end": "YYYY-MM-DD"},
       "fetched_at": "ISO timestamp"
   }
   ```

4. **Error handling:**
   - Wrap API call in try/except
   - Catch `googleapiclient.errors.HttpError`
   - For 400: Return `{"error": "Invalid video ID or bad request", "video_id": video_id}`
   - For 403: Return `{"error": "API quota exceeded or permission denied", "video_id": video_id}`
   - For no data: Return `{"error": "No retention data found for video", "video_id": video_id}`

5. **CLI interface:**
   ```python
   if __name__ == '__main__':
       import sys
       import json
       if len(sys.argv) < 2:
           print("Usage: python retention.py VIDEO_ID")
           sys.exit(1)
       result = get_retention_data(sys.argv[1])
       print(json.dumps(result, indent=2))
   ```

<task type="auto">
  <name>Task 2: Add drop-off point detection</name>
  <files>tools/youtube-analytics/retention.py</files>
  <action>
Add automatic drop-off detection to identify where viewers leave:

1. **Add function `find_drop_off_points(data_points, threshold=0.05)`:**
   - Scan consecutive data points
   - Flag "drop-off" when retention drops by more than threshold (5%) between adjacent points
   - Calculate the delta: `current.retention - previous.retention`
   - If delta < -threshold, it's a drop-off

2. **Return structure for drop-offs:**
   ```python
   [
       {
           "position": 0.15,  # 15% into video
           "retention_before": 0.72,
           "retention_after": 0.62,
           "drop": 0.10,  # 10% of viewers left
           "timestamp_hint": "~early in video"  # Human-readable hint
       },
       ...
   ]
   ```

3. **Timestamp hints based on position:**
   - 0.0-0.10: "intro"
   - 0.10-0.25: "early"
   - 0.25-0.50: "first half"
   - 0.50-0.75: "second half"
   - 0.75-0.90: "toward end"
   - 0.90-1.0: "conclusion"

4. **Update get_retention_data() to include drop-offs:**
   - Add `"drop_off_points": find_drop_off_points(data_points)` to return dict

5. **Optional CLI flag `--threshold`:**
   - Allow `python retention.py VIDEO_ID --threshold 0.03` for finer detection
   - Default remains 0.05 (5%)
