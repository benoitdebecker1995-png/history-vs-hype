<task type="auto">
  <name>Task 1: Create ctr.py with fallback</name>
  <files>tools/youtube-analytics/ctr.py</files>
  <action>
Create `tools/youtube-analytics/ctr.py` with:

1. **Import auth module:**
   ```python
   from auth import get_authenticated_service
   ```

2. **Main function `get_ctr_metrics(video_id, start_date=None, end_date=None)`:**
   - Default date range: 2020-01-01 to today
   - **TRY API FIRST:** Query YouTube Analytics API with:
     - `ids='channel==MINE'`
     - `metrics='views,videoThumbnailImpressions,videoThumbnailImpressionsClickRate'`
     - `dimensions='video'`
     - `filters=f'video=={video_id}'`

3. **Fallback logic (per research doc):**
   - If API returns data: return it
   - If API returns 400/403 for CTR metrics specifically:
     - Log "CTR not available via API for this video"
     - Return with `"ctr_available": False` and `"ctr_source": "api_unavailable"`
   - Do NOT crash or prompt for manual input (that's for Phase 9 analysis command)

4. **Return structure:**
   ```python
   # Success case:
   {
       "video_id": "...",
       "impressions": int,
       "ctr_percent": float,  # Already as percentage (e.g., 4.2 not 0.042)
       "ctr_available": True,
       "ctr_source": "api",
       "date_range": {"start": "YYYY-MM-DD", "end": "YYYY-MM-DD"},
       "fetched_at": "ISO timestamp"
   }

   # Fallback case:
   {
       "video_id": "...",
       "impressions": None,
       "ctr_percent": None,
       "ctr_available": False,
       "ctr_source": "api_unavailable",
       "note": "CTR metrics not available via API. Check YouTube Studio manually.",
       "date_range": {"start": "YYYY-MM-DD", "end": "YYYY-MM-DD"},
       "fetched_at": "ISO timestamp"
   }
   ```

5. **Error handling:**
   - For general 400: Return error dict
   - For 403 quota: Return error dict
   - For CTR-specific unavailability: Return fallback structure (not error)

6. **CLI interface:**
   ```python
   if __name__ == '__main__':
       import sys
       import json
       if len(sys.argv) < 2:
           print("Usage: python ctr.py VIDEO_ID")
           sys.exit(1)
       result = get_ctr_metrics(sys.argv[1])
       print(json.dumps(result, indent=2))
   ```

Either outcome is valid — script should not crash.
  </verify>
  <done>
Script attempts CTR fetch via API.
If unavailable, returns structured fallback (not error) with clear note.
  </done>

<task type="auto">
  <name>Task 2: Create video_report.py combined reporter</name>
  <files>tools/youtube-analytics/video_report.py</files>
  <action>
Create `tools/youtube-analytics/video_report.py` that orchestrates all three data scripts:

1. **Imports:**
   ```python
   from metrics import get_video_metrics
   from retention import get_retention_data
   from ctr import get_ctr_metrics
   ```

2. **Main function `generate_video_report(video_id)`:**
   - Call all three data functions
   - Combine into unified report structure
   - Handle if any individual call returns error

3. **Return structure:**
   ```python
   {
       "video_id": "...",
       "title": "...",  # From metrics
       "fetched_at": "ISO timestamp",

       "engagement": {
           "views": int,
           "watch_time_minutes": float,
           "avg_view_duration_seconds": int,
           "likes": int,
           "comments": int,
           "shares": int,
           "subscribers_gained": int
       },

       "ctr": {
           "impressions": int or None,
           "ctr_percent": float or None,
           "available": bool,
           "source": "api" or "api_unavailable"
       },

       "retention": {
           "avg_retention": float,
           "final_retention": float,
           "drop_off_points": [...],
           "data_points_count": int
       },

       "errors": []  # Any errors from individual calls
   }
   ```

4. **Error aggregation:**
   - If metrics call fails: Add to errors, set engagement to null
   - If retention call fails: Add to errors, set retention to null
   - If CTR call fails: Add to errors, set ctr to null
   - Return partial report even if some data unavailable

5. **CLI with output formats:**
   ```python
   if __name__ == '__main__':
       import sys
       import json
       if len(sys.argv) < 2:
           print("Usage: python video_report.py VIDEO_ID [--json|--markdown]")
           sys.exit(1)

       video_id = sys.argv[1]
       output_format = sys.argv[2] if len(sys.argv) > 2 else '--json'

       report = generate_video_report(video_id)

       if output_format == '--markdown':
           print(format_as_markdown(report))
       else:
           print(json.dumps(report, indent=2))
   ```

6. **Add `format_as_markdown(report)` helper:**
   - Human-readable format per research doc template
   - Include title, metrics table, retention summary, drop-off points
   - Mark CTR as "N/A (check YouTube Studio)" if unavailable
