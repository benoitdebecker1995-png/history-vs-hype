  <files>tools/youtube-analytics/variants.py</files>
  <action>
Create `tools/youtube-analytics/variants.py` -- a standalone CLI module for variant management. This is the user-facing tool for Phase 29.

**Module structure:**

1. **Imports and setup:**
   - Add tools/discovery to sys.path (matching analyze.py pattern on line 55-56)
   - Import KeywordDB from database
   - Import imagehash and PIL with graceful fallback (try/except ImportError with IMAGEHASH_AVAILABLE flag)
   - Import argparse for CLI

2. **Helper functions:**

   `generate_thumbnail_hash(file_path: str) -> Optional[str]`
   - Validate file exists with Path.exists()
   - If IMAGEHASH_AVAILABLE: open with PIL, run imagehash.phash(), return str(hash_value)
   - If not available: print warning "imagehash not installed, skipping hash generation", return None
   - On any error: return None with warning message

   `validate_video_id(video_id: str) -> str`
   - Strip whitespace
   - Check matches pattern r'^[\w-]{11}$' (same as analyze.py line 233)
   - Return video_id or raise ValueError

   `validate_tags(tags_str: str) -> List[str]`
   - Split by comma, strip whitespace from each
   - Filter empty strings
   - Return list of tag strings

3. **Command functions (each takes parsed args):**

   `cmd_register_thumb(args)`
   - Validate video_id, variant_letter (uppercase, single char A-Z)
   - Validate file_path exists (Path.exists()), print error and exit if not found
   - Parse visual_patterns from comma-separated string (e.g., "map,face,text")
   - Generate perceptual hash via generate_thumbnail_hash()
   - Call db.add_thumbnail_variant()
   - Print result: "Registered thumbnail variant {letter} for {video_id}" with hash if available
   - Suggested tags to show in help: map, face, text, document, evidence, split

   `cmd_register_title(args)`
   - Validate video_id, variant_letter
   - Parse formula_tags from comma-separated string
   - Call db.add_title_variant()
   - Print result: "Registered title variant {letter}: '{title}' ({len} chars)" with tags
   - Suggested tags to show in help: mechanism, document, paradox, question, comparison

   `cmd_record_ctr(args)`
   - Validate video_id
   - Accept ctr_percent, impression_count, view_count as required args
   - Accept optional --thumbnail-id and --title-id (integer IDs of active variants)
   - Accept optional --date (snapshot date, defaults to today)
   - Accept optional --late flag (marks as late entry)
   - Call db.add_ctr_snapshot()
   - Print result: "Recorded CTR snapshot: {ctr}% ({impressions} impressions, {views} views)"

   `cmd_list_variants(args)`
   - Accept video_id
   - Call db.get_thumbnail_variants(), db.get_title_variants(), db.get_ctr_snapshots()
   - Format as readable table output:
     - "Thumbnails:" header, then variant letter, file path, tags, hash (truncated to 8 chars)
     - "Titles:" header, then variant letter, title text (truncated to 50 chars), char count, tags
     - "CTR Snapshots:" header, then date, CTR%, impressions, views, active variant IDs
   - If no data: print "No variants registered for {video_id}"

   `cmd_show_snapshots(args)`
   - Accept video_id
   - Call db.get_ctr_snapshots()
   - Format as detailed table with all snapshot data
   - Show trend: compare latest to earliest CTR (if multiple snapshots exist)

4. **CLI entry point (if __name__ == '__main__'):**
   - Use argparse with subcommands:
     - `register-thumb VIDEO_ID VARIANT_LETTER FILE_PATH --tags "map,face"`
     - `register-title VIDEO_ID VARIANT_LETTER TITLE_TEXT --tags "mechanism,paradox"`
     - `record-ctr VIDEO_ID CTR IMPRESSIONS VIEWS [--thumbnail-id N] [--title-id N] [--date YYYY-MM-DD] [--late]`
     - `list VIDEO_ID`
     - `snapshots VIDEO_ID`
   - Add usage examples in description
   - Print help if no subcommand provided

**Docstring format:** Match analyze.py docstring style (module docstring with Usage, Python, Output sections).

**Error handling:** All commands wrap in try/except, print user-friendly error messages, exit with code 1 on failure.
