**Imports:**

- re for pattern detection
- Import KeywordDB from discovery/database (with try/except, feature flag SCORER_AVAILABLE)
- Import load_voice_patterns from section_diagnostics (with try/except fallback)

**Functions to implement:**

1. `count_evidence_markers(text)` -> int:
   - Count occurrences of: "according to", "page ", "in his", "in her", "the treaty states", "the document shows", direct quotes (text between quotation marks using regex)
   - Return total count

2. `measure_modern_relevance_gap(text)` -> int:
   - Markers: "today", "2024", "2025", "2026", "currently", "modern", "now", "recent", "still"
   - Find last occurrence of any marker, count words after it
   - If no marker found, return total word count
   - If marker is in last word, return 0

3. `detect_voice_patterns(text)` -> list[str]:
   - Check for: causal connectors ("consequently", "thereby", "which meant that"), evidence markers (overlap with count_evidence_markers), rhythm patterns (short fragments after long sentences -- use regex for sentence < 6 words after sentence > 20 words), question patterns ("?")
   - Use substring matching (same approach as section_diagnostics.py)
   - Return list of detected pattern names

4. `get_topic_baseline(topic_type)` -> dict:
   - Query KeywordDB for video_performance rows where topic_type matches
   - Calculate averages from lessons_learned JSON (if available) or video-level metrics
   - Fall back to channel average (all topic types) when topic has <3 videos
   - Fall back to hardcoded defaults when no data: {avg_section_length: 150, std_dev_length: 50, avg_evidence_density: 0.5, avg_modern_relevance_gap: 100, high_retention_patterns: [], video_count: 0, confidence: 'default'}
   - Return dict with: avg_section_length, std_dev_length, avg_evidence_density, avg_modern_relevance_gap, high_retention_patterns, video_count, confidence

5. `score_section(section_text, section_type, topic_type, baseline=None)` -> dict:
   - Calculate metrics: word_count, evidence_density (markers per 100 words), modern_relevance_gap, voice_patterns_found
   - Get baseline if not passed
   - Calculate deviation scores:
     - length_deviation = max(0, (word_count - baseline['avg_section_length']) / baseline['std_dev_length']) if std_dev > 0 else 0
     - evidence_penalty = max(0, baseline['avg_evidence_density'] - evidence_density)
     - relevance_penalty = max(0, modern_relevance_gap - 100) / 50
     - pattern_bonus = 0.05 per detected voice pattern (cap at 0.2)
   - Composite: raw = 1.0 - (length_deviation * 0.25 + evidence_penalty * 0.25 + relevance_penalty * 0.3) + pattern_bonus, clamped 0.0-1.0
   - Generate warnings:
     - Length > baseline + 1.5 * std_dev -> length warning
     - Evidence density < 0.3 -> evidence warning
     - Modern relevance gap > 150 words -> relevance warning
     - Intro starting with "The concept"/"The idea"/"The notion" -> abstract opening warning
     - Conclusion without "today"/"future"/"still"/"continues" -> weak closing warning
   - Risk: score < 0.5 = HIGH, 0.5-0.7 = MEDIUM, > 0.7 = LOW
   - Return: {score, risk_level, warnings, metrics}

6. `score_all_sections(sections, topic_type)` -> list[dict]:
   - Get baseline once, pass to each score_section call
   - Each section object expected to have .heading and .text attributes
   - Add 'section_heading' field to each result
   - Return list of score dicts

7. `format_retention_warnings(scored_sections)` -> str:
   - Markdown table: Section | Risk | Score | Top Warning
   - Only show MEDIUM and HIGH risk sections
   - If no risky sections, return "All sections scored LOW risk."
