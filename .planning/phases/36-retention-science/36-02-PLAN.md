  <behavior>
Score script sections for predicted retention risk based on historical patterns.

**Input/Output contracts:**

1. `score_section(section_text, section_type, topic_type, baseline=None)` -> dict
   - Input: section text (str), section_type ('intro'|'body'|'conclusion'), topic_type ('territorial'|'ideological'|'legal'|'colonial'|'general'), optional baseline dict
   - Output: {'score': float 0.0-1.0, 'risk_level': 'LOW'|'MEDIUM'|'HIGH', 'warnings': [{'issue': str, 'recommendation': str, 'pattern_ref': str}], 'metrics': {'word_count': int, 'evidence_density': float, 'modern_relevance_gap': int, 'voice_patterns_found': list}}

   Test cases:
   - Short section (50 words, body, territorial) with evidence and modern mention -> score >= 0.7, risk LOW
   - Long section (400 words, body, territorial) with no evidence, no modern mention -> score < 0.5, risk HIGH, warnings for length + evidence + relevance
   - Section with "consequently" and "according to" and "today" -> fewer warnings than section without any
   - Intro section with abstract opening ("The concept of...") -> warning for abstract opening
   - Conclusion with no forward-looking statement -> warning for weak closing

2. `get_topic_baseline(topic_type)` -> dict
   - Input: topic_type string
   - Output: {'avg_section_length': int, 'std_dev_length': float, 'avg_evidence_density': float, 'avg_modern_relevance_gap': int, 'high_retention_patterns': list, 'video_count': int, 'confidence': str}
   - When topic_type has <3 videos, return channel average baseline
   - When no data at all, return hardcoded defaults: {avg_section_length: 150, std_dev_length: 50, avg_evidence_density: 0.5, avg_modern_relevance_gap: 100, high_retention_patterns: [], video_count: 0, confidence: 'default'}

3. `score_all_sections(sections, topic_type)` -> list[dict]
   - Input: list of Section objects (from parser.py), topic_type
   - Output: list of score dicts (one per section), plus 'section_heading' field added to each

4. `format_retention_warnings(scored_sections)` -> str
   - Input: list of scored section dicts from score_all_sections
   - Output: markdown-formatted warnings report
   - Only shows MEDIUM and HIGH risk sections
   - Format: table header + rows with Section | Risk | Score | Top Warning

5. `count_evidence_markers(text)` -> int
   - Input: section text
   - Output: count of evidence markers ("according to", "page ", "in his", "in her", "the treaty states", "the document shows", direct quotes with quotation marks)
   - Used to calculate evidence_density = markers_per_100_words

6. `measure_modern_relevance_gap(text)` -> int
   - Input: section text
   - Output: word count since last modern relevance marker ("today", "2024", "2025", "2026", "currently", "modern", "now", "recent", "still")
   - If no marker found, return total word count (entire section is a gap)
   - If marker found at end, return 0

7. `detect_voice_patterns(text)` -> list[str]
   - Input: section text
   - Output: list of detected voice pattern names from Part 6
   - Detection via substring matching (same approach as section_diagnostics.py)
   - Check for: causal connectors, evidence markers, rhythm patterns (short fragments after long sentences), question patterns

**Scoring algorithm for score_section():**

1. Calculate section metrics:
   - word_count = len(text.split())
   - evidence_density = count_evidence_markers(text) / (word_count / 100) if word_count > 0 else 0
   - modern_relevance_gap = measure_modern_relevance_gap(text)
   - voice_patterns = detect_voice_patterns(text)

2. Get baseline (passed in or from get_topic_baseline)

3. Calculate deviation scores:
   - length_deviation = max(0, (word_count - baseline['avg_section_length']) / baseline['std_dev_length']) if std_dev > 0 else 0
   - evidence_penalty = max(0, baseline['avg_evidence_density'] - evidence_density)
   - relevance_penalty = max(0, modern_relevance_gap - 100) / 50  (penalty increases linearly after 100 words)
   - pattern_bonus = 0.05 per detected voice pattern (cap at 0.2)

4. Composite score:
   - raw = 1.0 - (length_deviation * 0.25 + evidence_penalty * 0.25 + relevance_penalty * 0.3) + pattern_bonus
   - score = clamp(raw, 0.0, 1.0)

5. Generate warnings for each detected issue:
   - Length > baseline + 1.5 * std_dev: "Section length X words exceeds {topic_type} avg {baseline}"
   - Evidence density < 0.3: "Low evidence density (X per 100 words) -- add quotes or source references"
   - Modern relevance gap > 150 words: "X words since modern relevance mention -- add 'which is why today...' bridge"
   - Section-type specific: intro with abstract opening, conclusion without forward statement

6. Risk level: score < 0.5 = HIGH, 0.5-0.7 = MEDIUM, > 0.7 = LOW

**get_topic_baseline():**
- Query KeywordDB for video_performance rows where topic_type matches
- Calculate averages from lessons_learned JSON (if available) or use video-level metrics
- Fall back to channel average (all topic types) when topic has <3 videos
- Fall back to hardcoded defaults when no data at all

**Error handling:**
- Return safe defaults on any error (score 0.5, risk MEDIUM, empty warnings)
- Never raise exceptions
- Follow error dict pattern for get_topic_baseline

**CLI (if __name__ == '__main__'):**
- `python retention_scorer.py SCRIPT_PATH [--topic TOPIC_TYPE]`
- Parse script with ScriptParser, score all sections, print formatted warnings
