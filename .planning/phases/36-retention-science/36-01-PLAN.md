  <name>Task 1: Create playbook_synthesizer.py with pattern extraction and Part 9 generation</name>
  <files>tools/youtube-analytics/playbook_synthesizer.py</files>
  <action>
Create tools/youtube-analytics/playbook_synthesizer.py with the following functions:

**Imports and setup:**
- Import sys, json, pathlib, statistics (mean, stdev), datetime
- Add discovery to path for KeywordDB import (same pattern as feedback_queries.py line 31-32)
- Import load_voice_patterns from section_diagnostics (for pattern effectiveness ranking)
- Feature flag: PLAYBOOK_AVAILABLE = True (set False on import errors)

**Core functions:**

1. `get_all_video_retention_data()` -> list[dict]:
   - Query KeywordDB for all videos in video_performance table that have retention data (biggest_drop_position IS NOT NULL or lessons_learned IS NOT NULL)
   - For each video, return: video_id, title, topic_type, avg_retention, biggest_drop_position, lessons_learned (JSON-parsed), conversion_rate
   - Return empty list on error (error dict pattern)
   - Minimum: needs at least 3 videos to be useful, return {'error': 'Insufficient data'} if fewer

2. `extract_opening_patterns(videos)` -> dict:
   - Analyze first-section retention data across videos
   - Group by topic_type (territorial, ideological, legal, colonial, general)
   - For each topic_type: calculate avg retention at 10% position (first ~60 seconds)
   - Identify which opening patterns (from section_diagnostics.load_voice_patterns()['openings']) correlate with higher retention
   - Return: {topic_type: {avg_opening_retention, recommended_patterns, confidence, video_count}}

3. `calculate_pacing_thresholds(videos)` -> dict:
   - From lessons_learned observations, extract pacing-related insights
   - Calculate per topic_type: avg biggest_drop_position, avg section length where drops occur
   - Determine thresholds: "sections over X words tend to drop" based on mean + 1 std dev
   - Fall back to channel average when topic_type has <3 videos
   - Return: {topic_type: {avg_section_length, max_section_before_drop, pattern_interrupt_interval, confidence, video_count}}

4. `extract_modern_relevance_rules(videos)` -> dict:
   - From lessons_learned, count how often "modern relevance" or "connection to present" appears in observations/actionable items
   - Calculate average drop position for videos where modern relevance was flagged as an issue
   - Derive gap tolerance per topic_type (words/time since last modern mention before drop)
   - Return: {topic_type: {max_gap_words, recommended_bridge_interval, confidence}}

5. `rank_voice_patterns(videos)` -> list[dict]:
   - Cross-reference video retention with which voice patterns were recommended by section_diagnostics
   - Rank patterns by frequency of recommendation (most recommended = most common gap)
   - Also identify which patterns are associated with high-retention videos (>= channel average)
   - Return: [{pattern_name, pattern_ref, times_recommended, avg_retention_when_present, effectiveness}]

6. `calculate_confidence(video_count, min_videos=3)` -> str:
   - <3 videos: 'insufficient' (don't generate rules)
   - 3-5 videos: 'low'
   - 6-10 videos: 'medium'
   - 11+ videos: 'high'

7. `synthesize_part9(min_confidence_videos=3)` -> str:
   - Orchestrate all extraction functions
   - Build Part 9 markdown with these sections:
     - Header with generation date, video count, confidence level
     - 9.1: Opening Retention Rules (per topic_type)
     - 9.2: Section Pacing Guidelines (length thresholds, pattern interrupt intervals)
     - 9.3: Modern Relevance Proximity Rules (gap tolerance per topic_type)
     - 9.4: Voice Pattern Effectiveness Ranking (which Part 6 patterns work best)
     - 9.5: Topic-Type Retention Baselines (table: topic_type | avg_retention | avg_drop_position | video_count | confidence)
     - 9.6: Anti-Pattern Summary (most common root causes from section_diagnostics)
   - Include confidence flags: "LOW confidence (N videos)" warnings inline
   - Include "Last synthesized: YYYY-MM-DD from N videos" header
   - Return markdown string

8. `write_part9_to_style_guide(part9_text)` -> dict:
   - Read .claude/REFERENCE/STYLE-GUIDE.md
   - Find existing "## Part 9:" section and replace it, OR append after Part 7 section if not found
   - IMPORTANT: Do NOT insert before Part 8 placeholder or renumber any parts. Part 8 is reserved for Phase 37. If Part 8 doesn't exist yet, append Part 9 after Part 7
   - Write updated file
   - Return {'status': 'updated', 'path': str} or {'error': msg}

**CLI entry point (if __name__ == '__main__'):**
- `python playbook_synthesizer.py` -- synthesize and print Part 9 to stdout (dry run)
- `python playbook_synthesizer.py --update` -- synthesize and write to STYLE-GUIDE.md
- `python playbook_synthesizer.py --json` -- output raw pattern data as JSON (for debugging)
- Use argparse for CLI

**Error handling:**
- Follow error dict pattern: return {'error': msg}, never raise
- Graceful degradation: if database has no retention data, print warning and exit cleanly
- If KeywordDB import fails, set PLAYBOOK_AVAILABLE = False

**Style notes:**
- Match existing codebase patterns (see feedback_queries.py for import pattern, section_diagnostics.py for voice pattern usage)
- Docstrings for all public functions
- Type hints for function signatures
