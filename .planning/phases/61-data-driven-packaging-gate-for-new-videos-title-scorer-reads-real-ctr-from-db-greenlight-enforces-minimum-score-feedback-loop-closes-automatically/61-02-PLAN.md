    ```python
    def ingest_synthesis_ctr(synthesis_path: Path, db: KeywordDB) -> dict:
        """
        Read CROSS-VIDEO-SYNTHESIS.md master table and store CTR to keywords.db.

        Returns {'written': int, 'skipped': int, 'unmatched': int, 'errors': list}
        """
    ```

    Implementation steps:
    1. Call _parse_synthesis_table(synthesis_path) to get video list
    2. For each video with ctr not None:
       a. Look up video_id by matching title against video_performance table
       b. Query: SELECT video_id FROM video_performance WHERE title LIKE '%{title_fragment}%'
          Use the first 40 chars of the title for matching (titles may have slight formatting differences)
       c. If no match found, increment unmatched counter and continue
       d. Call db.add_ctr_snapshot(video_id, ctr, impressions or 0, views or 0, snapshot_date='2026-02-23', is_late_entry=True)
       e. Track written/error counts
    3. Return summary dict

    Add CLI with argparse:

    ```
    python -m tools.ctr_ingest --synthesis path/to/file.md  # Custom synthesis path
    python -m tools.ctr_ingest --dry-run                    # Show what would be written
    ```

    Use tools.logging_config.get_logger for logging. Print summary to stdout on completion.

    IMPORTANT: The title matching between synthesis table and video_performance is the trickiest part. Use case-insensitive LIKE matching on a substantial substring of the title. Log any unmatched titles at WARNING level so the user knows which videos were skipped.
