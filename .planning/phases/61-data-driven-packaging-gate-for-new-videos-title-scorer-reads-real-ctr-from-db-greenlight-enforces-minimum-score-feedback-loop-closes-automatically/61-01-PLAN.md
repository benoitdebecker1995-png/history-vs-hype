    Then implement tools/title_ctr_store.py with a single public function:

    ```python
    def get_pattern_ctr_from_db(db_path: str, min_sample: int = 3) -> dict:
    ```

    Implementation:
    1. Open sqlite3 connection to db_path (read-only)
    2. Query: JOIN ctr_snapshots cs ON video_performance vp WHERE cs.video_id = vp.video_id AND cs.ctr_percent > 0 AND vp.title IS NOT NULL
    3. For each video, use only the latest snapshot (MAX snapshot_date per video_id WHERE ctr_percent > 0)
    4. Import detect_pattern from tools.title_scorer and classify each title
    5. Group by pattern, compute average ctr_percent per pattern
    6. Filter patterns where count < min_sample
    7. Convert CTR% to 0-100 score: score = min(100, max(0, int(ctr * 17)))
       - This calibration: 3.8% CTR = 64.6 (rounds to 65, matching current declarative baseline)
    8. Return dict mapping pattern -> score

    Use tools.logging_config.get_logger for logging. Handle DB errors gracefully (return empty dict on any sqlite3.Error).

    AVOID circular imports: title_ctr_store imports detect_pattern from title_scorer. title_scorer will import get_pattern_ctr_from_db from title_ctr_store. This is safe because they are leaf-level imports with no cycles — title_ctr_store does NOT import score_title.
  </action>
