<objective>
Build the surprise clause detection module: a modular post-translation analysis pass that compares each translated clause against a user-provided narrative baseline, classifying surprises by severity and generating script suggestions.

Purpose: This is the "aha moment" generator for the Untranslated Evidence format. When a translated clause directly contradicts what English-language sources commonly say about the document, that becomes a highlight moment in the video. The three-tier severity system (Minor, Notable, Major) maps directly to video pacing: Major surprises get the most screen time.

Output: surprise_detector.py module + CLI surprise subcommand

**`surprise_detector.py`** — `SurpriseDetector` class:

- `__init__(self, model: str = 'claude-sonnet-4-20250514')`: Initialize with anthropic client. Check for ANTHROPIC_API_KEY env var. Same pattern as Translator.
- `detect_surprises(self, sections: List[Dict], narrative: str, source_language: str, document_context: Optional[str] = None, on_progress: Optional[Callable] = None) -> Dict[str, Any]`: Main entry. Analyze all translated sections against narrative.
  - `narrative` is user-provided text describing what people commonly believe the document says. Example: "The Statut des Juifs is commonly described as defining Jews by religion. Most English sources say it banned Jews from government jobs. Wikipedia describes it as targeting Jewish refugees."
  - For each section, call `_analyze_clause` with the narrative + full translated document context
  - Return: `{'surprises': List[Dict], 'total_clauses': int, 'surprise_count': int, 'by_severity': Dict[str, int], 'narrative_used': str}`
  - `by_severity`: `{'major': int, 'notable': int, 'minor': int}`

- `_analyze_clause(self, clause_id: str, original: str, translation: str, narrative: str, full_context: str, source_language: str, document_context: Optional[str] = None) -> Dict[str, Any]`: Analyze a single clause against narrative.
  - Build Claude prompt:
    - System: "You are a historical document analyst specializing in comparing primary source translations against popular narratives. Your task is to identify where the actual document text contradicts, complicates, or adds nuance missing from common English-language descriptions of this document."
    - Prompt includes:
      1. The narrative baseline (what people commonly believe)
      2. The specific clause (original + translation)
      3. Full document context for reference
      4. Instructions: "Analyze whether this clause contradicts, complicates, or is entirely absent from the common narrative. Classify as: MAJOR (directly contradicts common narrative), NOTABLE (significant omission — people don't know this clause exists and it changes the picture), MINOR (adds nuance but doesn't fundamentally change the narrative), or NONE (clause aligns with common narrative). For non-NONE results, write a 1-2 sentence script suggestion for how to present this surprise moment in a video. Respond in JSON."
    - Expected JSON: `{"severity": "major"|"notable"|"minor"|"none", "explanation": str, "what_people_think": str, "what_document_says": str, "script_beat": str|null}`
  - Parse Claude's JSON response
  - Return: `{'clause_id': str, 'severity': str, 'explanation': str, 'what_people_think': str, 'what_document_says': str, 'script_beat': str, 'original': str, 'translation': str}` for non-NONE severities
  - Return `{'clause_id': str, 'severity': 'none'}` for clauses that align with narrative
  - Return `{'error': msg}` on API failure

- `format_report(self, results: Dict) -> str`: Format surprise detection as markdown report.
  - Summary at top:
    ```
    # Surprise Clause Analysis

    **Narrative tested:** [first 200 chars of narrative...]
    **Clauses analyzed:** X
    **Surprises found:** Y (Z major, W notable, V minor)
    ```
  - Group by severity (Major first, then Notable, then Minor):
    ```
    ## MAJOR Surprises

    ### Article 2 — MAJOR
    **Common belief:** [what_people_think]
    **Document actually says:** [what_document_says]
    **Why this matters:** [explanation]
    **Script suggestion:** [script_beat]

    > Original: [original text]
    > Translation: [translation text]
    ```
  - At end, list clauses with no surprises in a compact list: "No surprises: Articles 1, 5, 8"

- `update_sections_with_surprises(self, sections: List[Dict], surprises: List[Dict]) -> List[Dict]`: Merge surprise data into section dicts.
  - Add `'surprise'` key to each section dict that has a surprise: `{'severity': str, 'explanation': str, 'script_beat': str}`
  - Sections without surprises get `'surprise': None`
  - Returns updated sections list (for feeding into formatter)
