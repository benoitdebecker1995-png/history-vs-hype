**`cross_checker.py`** — `CrossChecker` class:

- `__init__(self, deepl_api_key: Optional[str] = None)`: Initialize. Check for DEEPL_AUTH_KEY env var (or accept via parameter). Store key if available. Try importing googletrans (or deep_translator) as fallback — use try/except, don't require either.
- `_get_available_backends(self) -> List[str]`: Return list of available backends. Priority order:
  1. "deepl" — if API key available
  2. "googletrans" — if googletrans library installed (`from googletrans import Translator as GoogleTranslator`)
  3. "deep_translator" — if deep-translator library installed (`from deep_translator import GoogleTranslator as DeepTranslatorGoogle`)
  - If none available, return empty list (cross-checking will be skipped with clear message)
- `_translate_with_backend(self, text: str, source_lang: str, target_lang: str = 'en', backend: str = 'deepl') -> Dict[str, Any]`: Translate text using specified backend.
  - DeepL: Use requests to call `https://api-free.deepl.com/v2/translate` (or `api.deepl.com` for paid key). Map language codes (e.g., 'french' -> 'FR', 'spanish' -> 'ES', 'german' -> 'DE').
  - googletrans: Use `GoogleTranslator().translate(text, src=src_code, dest='en')`.
  - deep_translator: Use `DeepTranslatorGoogle(source=src_code, target='en').translate(text)`.
  - Return: `{'translation': str, 'backend': str}` or `{'error': msg}`
- `check_clause(self, original: str, claude_translation: str, source_language: str, clause_id: str) -> Dict[str, Any]`: Cross-check a single clause.
  1. Get independent translation from best available backend
  2. Compare Claude vs independent translation using Claude API call with semantic comparison prompt:
     - System: "You are a translation quality assessor. Compare two translations of the same text. Identify SEMANTIC differences only — changes in meaning, missing information, altered legal implications. Ignore stylistic differences (shall/will, may/can when synonymous, word order variations that don't change meaning)."
     - Prompt: "Original ({language}): {original}\nTranslation A (Claude): {claude}\nTranslation B ({backend}): {independent}\n\nAre there semantic differences? Respond in JSON: {\"has_discrepancy\": bool, \"severity\": \"none\"|\"minor\"|\"significant\", \"explanation\": str, \"recommendation\": str}"
  3. Parse Claude's JSON response
  - Return: `{'clause_id': str, 'has_discrepancy': bool, 'severity': str, 'claude_translation': str, 'independent_translation': str, 'backend': str, 'explanation': str, 'recommendation': str}`
  - Return `{'error': msg}` on failure
- `check_document(self, sections: List[Dict], source_language: str, on_progress: Optional[Callable] = None) -> Dict[str, Any]`: Cross-check all translated sections.
  - Check available backends first. If none: return `{'error': 'No cross-check backends available. Install googletrans (pip install googletrans==4.0.0-rc1) or set DEEPL_AUTH_KEY.', 'skipped': True}`
  - Iterate sections, call check_clause for each
  - Collect inline results per clause + build summary
  - Return: `{'results': List[Dict], 'summary': Dict, 'backend_used': str}`
  - Summary: `{'total_clauses': int, 'discrepancies_found': int, 'significant_count': int, 'minor_count': int}`
- `format_report(self, check_results: Dict) -> str`: Format cross-check results as markdown.
  - Summary section at top: "Cross-Check Summary: X/Y clauses have discrepancies (Z significant)"
  - Per-clause inline flags for clauses with discrepancies:
    ```
    ### Article 1 — DISCREPANCY (significant)
    **Claude:** [claude translation]
    **{backend}:** [independent translation]
    **Difference:** [1-2 sentence explanation]
    **Recommendation:** [what to do]
    ```
  - Clauses without discrepancies get a simple checkmark line: "Article 2 — No discrepancy"

**`legal_annotator.py`** — `LegalAnnotator` class:

- `__init__(self, model: str = 'claude-sonnet-4-20250514')`: Initialize with anthropic client (same pattern as Translator). Check for ANTHROPIC_API_KEY.
- `annotate_clause(self, original: str, translation: str, source_language: str, clause_id: str, document_context: Optional[str] = None) -> Dict[str, Any]`: Annotate legal/technical terms in a single clause.
  - Send to Claude with prompt:
    - System: "You are a legal/historical terminology expert. Analyze the original-language legal text and its English translation. Identify terms that have NO direct English equivalent — terms where the translation is an approximation. For each such term: provide the original term, the dictionary definition in the source language, the closest English equivalent used, alternative English renderings, and whether this term is commonly rendered DIFFERENTLY in English-language discussions of similar documents (mistranslation flag). If the term had a different meaning in the historical period vs modern usage, note both. Only annotate terms with genuine translation difficulty — not every legal term. Respond in JSON."
    - Expected JSON response: `{"annotations": [{"original_term": str, "source_language_definition": str, "english_equivalent": str, "alternatives": [str], "historical_context": str|null, "commonly_mistranslated": bool, "common_mistranslation": str|null, "note": str}]}`
  - Return: `{'clause_id': str, 'annotations': List[Dict], 'annotation_count': int}`
  - Return `{'error': msg}` on failure
- `annotate_document(self, sections: List[Dict], source_language: str, document_context: Optional[str] = None, on_progress: Optional[Callable] = None) -> Dict[str, Any]`: Annotate all translated sections.
  - Iterate sections, call annotate_clause for each
  - Collect all annotations
  - Return: `{'sections': List[Dict], 'total_annotations': int, 'mistranslation_flags': int}`
  - Each section dict gets `'footnotes'` field added: list of formatted footnote strings
- `format_footnotes(self, annotations: List[Dict]) -> List[str]`: Convert annotation dicts to numbered footnote strings.
  - Format per footnote:
    ```
    **[term]** (original: [original_term]): [source_language_definition]. English equivalent: "[english_equivalent]". Alternatives: [alternatives joined]. [If historical_context:] Historical note: [historical_context]. [If commonly_mistranslated:] WARNING: Commonly rendered as "[common_mistranslation]" in English-language sources — this is misleading because [note].
    ```
  - Only produce footnotes for terms with genuine annotation value (filter out empty/trivial results)
