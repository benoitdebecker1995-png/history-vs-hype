**`__init__.py`**: Package init with version `"0.1.0"` and convenience imports for StructureDetector, Translator, Formatter.

**`structure_detector.py`** — `StructureDetector` class:

- `detect_structure(text: str, document_type: Optional[str] = None) -> Dict[str, Any]`: Main entry. Accepts raw document text, optionally a type hint (legal_code, treaty, decree, book, letter — same types as Phase 39 structure_assessor).
- Auto-detection algorithm:
  1. Split text into lines, strip whitespace
  2. Look for article/section markers using regex patterns:
     - French legal: `Article \d+`, `Art\. \d+`, `ARTICLE \d+`
     - Spanish legal: `Artículo \d+`, `Art\. \d+`
     - German legal: `Artikel \d+`, `§ \d+`, `Art\. \d+`
     - Latin/treaty: `Articulus \d+`
     - Generic numbered: `Section \d+`, `Chapter \d+`, `Clause \d+`
     - Roman numerals: `^[IVXLCDM]+\.?\s` at line start
  3. If no article markers found, fall back to paragraph-based splitting (double newline boundaries)
  4. Extract preamble (text before first article marker) as separate section
  5. For each detected section: capture heading/marker text, body text, start/end line numbers
- Return: `{'document_type': str, 'detected_type': str, 'sections': List[Dict], 'section_count': int, 'has_preamble': bool, 'raw_text': str}`
  - Each section dict: `{'id': str, 'heading': str, 'body': str, 'start_line': int, 'end_line': int}`
  - `id` format: "preamble", "article-1", "article-2", etc.
  - `detected_type` is the auto-detected type; `document_type` is user override if provided
- Integration with Phase 39: If tools/document_discovery/structure_assessor.py exists, import its TIMING_ESTIMATES for video length estimation on detected structure. Use try/except for optional dependency — don't require Phase 39 to be installed.
- Return `{'error': msg}` on failure (empty text, no sections detected).

**`formatter.py`** — `Formatter` class:

- `format_paired(sections: List[Dict], output_format: str = 'markdown') -> str`: Main entry. Takes translated sections and produces paired output.
  - Each section dict expected: `{'id': str, 'heading': str, 'original': str, 'translation': str, 'footnotes': Optional[List[str]]}`
  - Markdown output format (clause-by-clause):
    ```
    ## Article 1: [heading]

    ### Original (French)
    > [original text, blockquoted]

    ### Translation
    [English translation text]

    [If footnotes exist:]
    ---
    **Notes:**
    1. [footnote text]
    2. [footnote text]
    ```
  - Each clause section is self-contained (ready for split-screen: left panel = original blockquote, right panel = translation)
- `format_summary(sections: List[Dict], metadata: Dict) -> str`: Generate summary header with document name, language, section count, translation date, and tool versions used.
- `format_json(sections: List[Dict], metadata: Dict) -> str`: JSON output for machine consumption. Return JSON string with `{'metadata': Dict, 'sections': List[Dict]}`.
- Return `{'error': msg}` on failure.

**`translator.py`** — `Translator` class:

- `__init__(self, model: str = 'claude-sonnet-4-20250514')`: Initialize with anthropic client. Check for ANTHROPIC_API_KEY env var. Store model name. Use anthropic SDK pattern from notebooklm_bridge.py (try/except import, error dict on missing key).
- `translate_clause(clause_text: str, full_document: str, source_language: str, clause_id: str, document_context: Optional[str] = None) -> Dict[str, Any]`: Translate a single clause with full document context.
  - Build prompt that includes:
    1. Full document text with the active clause highlighted: wrap active clause in `>>> TRANSLATE THIS CLAUSE <<<` markers
    2. Source language identification
    3. Instructions: "Translate the highlighted clause faithfully. Preserve legal/technical meaning. Do not paraphrase or simplify. If a term has no direct English equivalent, translate it as closely as possible and note the difficulty."
    4. Optional document_context (e.g., "This is a 1940 French statute defining who counts as Jewish under Vichy law")
  - System prompt: "You are a legal/historical document translator specializing in faithful clause-by-clause translation. Accuracy and precision over readability. Preserve the original's register and formality. Flag terms with no direct English equivalent."
  - Return: `{'clause_id': str, 'original': str, 'translation': str, 'notes': List[str], 'model': str}`
  - `notes` captures any translator observations (ambiguous terms, alternative renderings)
  - Return `{'error': msg}` on API failure
- `translate_document(sections: List[Dict], source_language: str, full_text: str, document_context: Optional[str] = None, on_progress: Optional[Callable] = None) -> Dict[str, Any]`: Translate all sections sequentially.
  - Iterate through sections from structure_detector output
  - Call translate_clause for each section, passing full_text as context
  - Call on_progress callback after each clause (for CLI progress display): `on_progress(current_index, total_count, clause_id)`
  - Assemble results into list of translated section dicts
  - Return: `{'sections': List[Dict], 'source_language': str, 'model': str, 'clause_count': int}`
  - Each section dict in output: `{'id': str, 'heading': str, 'original': str, 'translation': str, 'notes': List[str]}`

**`cli.py`** — CLI entry point with argparse:

Subcommands:
1. `detect` — Run structure detection only (preview before translating)
   - `python tools/translation/cli.py detect --file document.txt`
   - `python tools/translation/cli.py detect "pasted text here"`
   - `cat document.txt | python tools/translation/cli.py detect -` (stdin)
   - `--type TYPE` — override document type
   - Output: detected structure with section list and count
   - `--json` flag for machine-readable output

2. `translate` — Full translation pipeline
   - `python tools/translation/cli.py translate --file document.txt --language french`
   - `python tools/translation/cli.py translate "text" --language spanish`
   - `cat doc.txt | python tools/translation/cli.py translate - --language french`
   - `--language LANG` — REQUIRED, source language
   - `--type TYPE` — override document type detection
   - `--context "description"` — optional document context for better translation
   - `--output PATH` — write output to file (default: stdout)
   - `--format FORMAT` — output format: markdown (default) or json
   - `--dry-run` — detect structure and show plan without translating (shows sections to translate + estimated API calls)
   - Progress display: print clause count progress to stderr as each clause completes (e.g., "Translating clause 3/10...")

Input handling:
- If first positional arg is `-`, read from stdin
- If `--file PATH` provided, read from file
- Otherwise treat positional arg as literal text
- Validate: error if text is empty or too short (< 10 chars)

All output as formatted markdown to stdout (consistent with Phase 39 CLI). JSON via `--json` flag. Exit code 0 on success, 1 on error.
