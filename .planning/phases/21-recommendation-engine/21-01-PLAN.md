  <files>tools/discovery/recommender.py</files>
  <action>
Create `tools/discovery/recommender.py` with the following structure:

**Imports and path setup:**
- Add youtube-analytics to sys.path (same pattern as orchestrator.py)
- Import: KeywordDB from database, extract_winning_patterns from pattern_extractor
- Import: Path, argparse, json, datetime, typing

**Function 1: `get_existing_topics() -> List[str]`**
- Scan `video-projects/_IN_PRODUCTION/` and `video-projects/_ARCHIVED/`
- Extract topic slugs from folder names: `{number}-{topic-slug-year}` -> `topic slug`
- Handle variations: convert hyphens to spaces, strip year suffix
- Return lowercase list for case-insensitive matching
- Handle missing directories gracefully (return empty list, no crash)

**Function 2: `topic_matches_existing(keyword: str, existing: List[str]) -> bool`**
- Fuzzy match: check if any existing topic words appear in keyword
- Use word-level matching (not substring): "iran coup" matches "iran-1953-coup" but not "iranian"
- Return True if match found, False otherwise

**Function 3: `calculate_pattern_multiplier(keyword: str, topic_type: str, angles: List[str], patterns: dict) -> Tuple[float, List[str]]`**
- Takes keyword, its topic_type, angles, and winning patterns dict
- Returns (multiplier, reasons) tuple
- Multiplier logic:
  - +0.3 if topic_type matches patterns['top_converter_profile']['dominant_topic']
  - +0.2 if any angle in patterns['top_converter_profile']['dominant_angles']
  - +0.1 if topic_type correlates with channel strength > 70 (check patterns['channel_strengths'])
- Cap at 1.5 maximum multiplier
- Build reasons list explaining each boost

**Class: TopicRecommender**
- `__init__(self, db: KeywordDB)` - Store db reference
- `recommend(self, limit: int = 5, topic_filter: Optional[str] = None) -> Dict`
  1. Load winning patterns via extract_winning_patterns()
  2. Get all ANALYZED keywords via db.get_keywords_by_lifecycle('ANALYZED')
  3. Get existing topics via get_existing_topics()
  4. Filter out keywords matching existing topics
  5. For each remaining keyword:
     - Get opportunity_score_final and opportunity_category from database record
     - Classify topic_type and angles (use TAG_VOCABULARY or cached classification)
     - Calculate pattern_multiplier and reasons
     - final_score = opportunity_score * multiplier (capped at 100)
  6. Sort by final_score descending
  7. Apply topic_filter if provided
  8. Return top N with reasoning
- `format_recommendation(self, rec: dict) -> str` - Format single recommendation as markdown
- `format_report(self, recommendations: List[dict]) -> str` - Full markdown report

**CLI (main function):**
- argparse with:
  - `--limit N` (default 5)
  - `--json` (JSON output)
  - `--refresh` (force refresh patterns)
  - `--topic-type TYPE` (filter by topic)
- Print recommendations or JSON based on flags

**Output structure:**
```python
{
    'recommendations': [
        {
            'rank': 1,
            'keyword': 'treaty of versailles',
            'opportunity_score': 72.5,
            'pattern_multiplier': 1.3,
            'final_score': 94.3,
            'category': 'Excellent',
            'reasons': ['Matches dominant topic: territorial', 'Legal angle: +20%'],
            'action': 'python orchestrator.py "treaty of versailles" --report'
        }
    ],
    'patterns_used': {
        'dominant_topic': 'territorial',
        'dominant_angles': ['legal', 'historical'],
        'channel_strengths': {...}
    },
    'excluded_count': 15,  # How many topics filtered due to existing work
    'analyzed_at': '2026-02-02T...'
}
```

**Error handling:**
- If no ANALYZED keywords: return helpful message to run orchestrator first
- If pattern extraction fails: proceed with multiplier=1.0 for all (graceful degradation)
- If folder scan fails: log warning but continue (don't crash)
