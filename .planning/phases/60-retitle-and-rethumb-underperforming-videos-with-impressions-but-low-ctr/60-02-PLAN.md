**Mode 4: `/retitle --check [video-id]`**
Instructions for Claude to:
1. Find the POST-PUBLISH-ANALYSIS file for the video:
   - First check `channel-data/analyses/POST-PUBLISH-ANALYSIS-{video-id}.md`
   - Then check video-projects folders via VIDEO_PROJECT_MAP
2. Read the SWAP LOG section. If no SWAP LOG exists, tell user: "No swap recorded for this video. Run /retitle first to generate a swap, then add SWAP LOG after executing the swap in YouTube Studio."
3. Parse the most recent "pending" row to get swap date
4. Calculate days since swap. If < 7 days: refuse to measure. Print: "Swap was [N] days ago (on [date]). Check back on [date + 7 days] for reliable data. Older videos need 7 days for YouTube to re-test packaging."
5. If >= 7 days: ask user "What's the current CTR in YouTube Studio for [video title]? (Check Studio > Analytics > Reach)"
6. Compare pre-CTR (from SWAP LOG) with post-CTR (from user input):
   - If post-CTR >= pre-CTR + 0.5%: Mark as SUCCESS
     - Update SWAP LOG row: Post-CTR = user value, Result = `+X.X% SUCCESS`
     - Tell user to update CROSS-VIDEO-SYNTHESIS.md with new CTR
     - Run `python -m tools.ctr_ingest` to feed improved data into keywords.db
     - Print: "Title swap successful. CTR improved by +X.X%. Feedback loop closed — title_scorer scores updated."
   - If post-CTR < pre-CTR + 0.5%: Mark as FLAT/REVERTED
     - Update SWAP LOG row: Post-CTR = user value, Result = `flat REVERTED` or `-X.X% REVERTED`
     - Display old title for immediate copy-paste revert
     - Print: "Title swap did not improve CTR. Revert to: [old title]. Copy and paste into YouTube Studio."

**Mode 5: `/retitle --revert [video-id]`**
Instructions for Claude to:
1. Find POST-PUBLISH-ANALYSIS file (same search as --check)
2. Read SWAP LOG section
3. Find most recent swap entry
4. Display clearly:
   ```
   REVERT TITLE for [video-id]:
   Old title: [exact old title from SWAP LOG]
   Studio link: https://studio.youtube.com/video/[id]/edit

   Copy the old title above, paste in YouTube Studio, and save.
   ```
5. After user confirms revert, update SWAP LOG Result column to "REVERTED"
