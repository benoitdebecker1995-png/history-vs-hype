**Frontmatter:**
- description: "Retitle underperforming videos — audit, generate candidates, output swap checklist"
- model: sonnet

**Command structure — three modes:**

**Mode 1: `/retitle` (full pipeline, default)**
Instructions for Claude to:
1. Read `channel-data/patterns/CROSS-VIDEO-SYNTHESIS.md`
2. Run `python -m tools.retitle_audit --top 5 --min-impressions 500` to get ranked underperformers
3. Apply retention weighting to re-sort results: `priority_score = wasted_impressions * (1 + retention_bonus)` where retention_bonus is +0.5 if retention >= 35%, +0.25 if 25-35%, +0 below 25%. This surfaces content-quality wins trapped by bad packaging over videos with dual packaging+content issues.
4. For each of the top 5 videos:
   a. Run `generate_script_titles(video_id, current_title)` via `python -m tools.retitle_gen`
   b. If no script-generated titles: fall back to CANDIDATES dict in retitle_gen.py, then to `channel-data/RETITLE-RECOMMENDATIONS.md`
   c. Score all candidates via `score_title()` — block anything with grade REJECTED or score < 65
   d. Check thumbnail compliance: if VIDEO_PROJECT_MAP has a project folder, run `check_project(folder_path)`; otherwise note "THUMBNAIL: Manual check required — no metadata file found"
   e. Auto-suggest thumbnail map type based on content signals:
      - Two actors/conflict in title -> split-map
      - Movement/extraction verbs -> arrow-flow
      - Legal document/treaty central -> document-on-map
      - Multiple claimants (3+) -> labeled-zone
      - Default -> split-map
5. Generate `channel-data/SWAP-CHECKLIST.md` with the format specified in 60-RESEARCH.md:
   - Header with date, batch size, measurement date (today + 7 days)
   - Per video: Video ID, Studio link (`https://studio.youtube.com/video/[id]/edit`), OLD TITLE (verbatim for revert), NEW TITLE (best scoring candidate with score), NEW DESCRIPTION (first 3 lines reframed for new title), THUMBNAIL section (SWAP NEEDED vs COMPLIANT, map type, concept, color scheme), PRE-SWAP METRICS (CTR, impressions, retention)
   - Footer: Post-Swap Checklist with action items including "run `/retitle --check [id]` for each" on measurement date
6. Print summary table to user showing 5 videos, old vs new title, score, thumbnail status

**Mode 2: `/retitle --audit`**
Instructions for Claude to:
1. Run `python -m tools.retitle_audit --top 10 --min-impressions 500`
2. Apply retention weighting sort (same formula as full pipeline)
3. Display format_report() output with retention-weighted priority column
4. Do NOT generate candidates or create SWAP-CHECKLIST

**Mode 3: `/retitle [project-slug]` (single video)**
Instructions for Claude to:
1. Find video in CROSS-VIDEO-SYNTHESIS.md by project slug match
2. Run full pipeline for that single video only
3. Output results inline (no SWAP-CHECKLIST file for single video)

**Important instructions to include in the command file:**
- "NEVER suggest titles from generic topic knowledge. ALWAYS read the actual SRT/script and base ALL suggestions on specific content from the video." (per feedback_titles-from-scripts.md)
- "Score threshold: 65 minimum. REJECTED grade = hard block. Show scores for all candidates so user can compare."
- "SWAP-CHECKLIST.md is ephemeral — regenerated each run. SWAP LOG in POST-PUBLISH-ANALYSIS is the permanent record (see /retitle --check)."
- Reference tools/SWAP-PROTOCOL.md for swap timing guidance but note: "For retitles of existing videos, use 7-day measurement window (not 48h)"
