    1. `load(path=None)` — reads `channel-data/niche_benchmark.json` relative to project root. Returns full dict or None. No caching. Never raises.

    2. `_vps_to_score(vps: float) -> int` — converts VPS to 0-100 score. Calibration: declarative median 0.565 VPS -> ~65 score. Formula: `min(100, max(0, int(vps * 115)))`.

    3. `get_niche_score(pattern: str, data: dict = None) -> Optional[int]` — loads data if not passed, looks up `by_pattern[pattern]`, returns `_vps_to_score(median_vps)`. Returns None if:

       - pattern not in by_pattern
       - confidence == "LOW" AND sample_count < 3 (guards against versus n=1)

    4. `TOPIC_GRADE_THRESHOLDS` dict — per CONTEXT.md two-tier system:
       ```python
       TOPIC_GRADE_THRESHOLDS = {
           'territorial':          {'pass': 50, 'good': 65},
           'ideological':          {'pass': 60, 'good': 70},
           'political_fact_check': {'pass': 75, 'good': 85},
           'general':              {'pass': 60, 'good': 70},
       }
       ```

    5. `get_topic_thresholds(topic_type: str) -> dict` — returns thresholds for topic type, falling back to 'general'.

    6. `normalize_topic_type(t: str) -> str` — maps performance.py taxonomy to niche_benchmark.json taxonomy:
       - 'politician' -> 'political_fact_check'
       - 'colonial', 'legal' -> 'territorial'
       - 'archaeological', 'medieval' -> 'general'
       - passthrough for 'territorial', 'ideological', 'political_fact_check', 'general'

    IMPORTANT: Keep colon as HARD REJECT in title_scorer.py. Niche colon data (0.776 VPS) is inflated by pipe-style titles from Knowing Better/Kraut. Do NOT let niche data override the colon penalty. Document this in a comment.
