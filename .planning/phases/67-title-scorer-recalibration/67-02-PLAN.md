       Replace the per-title scoring loop to call `score_title()` from title_scorer.py instead:
       ```python
       from tools.title_scorer import score_title as ts_score_title
       ```
       For each title in `titles`, call `ts_score_title(title, db_path=db_path, topic_type=topic_type)` where:
       - `db_path`: resolve keywords.db path (same as title_scorer CLI does)
       - `topic_type`: passed through from `_score_title_metadata()`'s existing `topic_type` parameter

       Map the score_title() result dict to the preflight return format:
       - `result['score']` = best title's score (already the pattern)
       - `result['best_title']` = highest-scoring title
       - `result['title_pattern']` = best title's pattern
       - `result['issues']` = collect from hard_rejects + suggestions
       - `result['notes']` = collect niche context

    2. **Add niche context to notes:**
       When niche_percentile_label is non-empty:
       ```python
       result['notes'].append(f"Niche position: {niche_percentile_label}")
       ```
       When fallback_warning is present:
       ```python
       result['notes'].append(fallback_warning)
       ```
       When gap_message is present:
       ```python
       result['notes'].append(f"Topic target: {gap_message}")
       ```

    3. **Preserve the existing return dict shape** — `_score_title_metadata()` returns `{'score', 'best_title', 'predicted_ctr', 'formulas', 'title_pattern', 'issues', 'notes'}`. Do not change these keys. Only enrich the values within `notes` and `issues`.
