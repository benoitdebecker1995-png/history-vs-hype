Create `tools/youtube-analytics/transcript_analyzer.py` with stdlib-only implementation (re, pathlib, json, collections.Counter, datetime).

**Core functions:**

1. `read_transcript(file_path: Path) -> str` — Format-aware reader:
   - `.srt`: Strip index lines + SRT timestamps (`\d+\n\d{2}:\d{2}:\d{2},\d+ --> ...\n`) + HTML tags
   - `.vtt`: Strip WEBVTT header + VTT timestamps (`\d{2}:\d{2}:\d{2}\.\d+ --> ...`) + HTML tags
   - `.txt`: Read directly
   - Clean up triple+ newlines to double newlines
   - Handle encoding with `encoding='utf-8', errors='replace'`

2. `get_creator_name(file_path: Path) -> str` — Creator attribution:
   - File in subfolder → folder name (e.g., `transcripts/Kraut/file.txt` → "Kraut")
   - File in `transcripts/` root → "History vs Hype"
   - File in `niche-research/` subfolder → "niche-research" (keep as-is)

3. `get_first_n_words(text: str, n: int = 500) -> str` — Extract first N words (proxy for ~2 min opening)

4. `get_context(text: str, position: int, window: int = 100) -> str` — Extract text window around position

5. `extract_opening_hook(text: str) -> Dict[str, Any]` — Detect hook patterns in first 500 words:
   - `visual_contrast`: Pattern for "[Show X] → [Show Y] → [State tension]" (re: `\b(now|instead|but)\b.*\b(see|look|open|shows?)\b`)
   - `fact_check_declaration`: "I fact-checked", "I read", "I checked", "I found" (re: `\b(fact[- ]check|I checked|I read|I found)\b`)
   - `current_event`: "today", "yesterday", "last week", "this month", "in 2024", "in 2025" (re: `\b(today|yesterday|last (week|month)|this (week|month|year)|in 20[12]\d)\b`)
   - `escalation_timeline`: "X days later", "then", "after that", "within hours" (re: `\b(\d+ (days?|hours?|weeks?) later|within (hours|days|weeks))\b`)
   - `question_hook`: Opens with a question mark in first 100 words
   - Return: `{'category': 'opening_hook', 'detected_patterns': [...], 'text_sample': first_200_chars, 'confidence': 'high'|'low'}`

6. `extract_transitions(text: str) -> List[Dict]` — Find transition patterns throughout:
   - `causal_chain`: "consequently", "thereby", "which meant that", "this led to", "as a result", "because of this"
   - `temporal_jump`: "Now,", "Fast forward", "Years later", "Meanwhile", "By [year]"
   - `pivot_phrase`: "So how did", "But here's where", "And this is where", "Here's the problem"
   - `contrast_shift`: "But", "However", "On the other hand", "And yet" (at sentence start only)
   - Return list of `{'pattern_type': str, 'text': context_window, 'position': int}`

7. `extract_evidence_patterns(text: str) -> Dict[str, Any]` — Count evidence presentation:
   - `direct_quotes`: Matches like `"Here's what .{1,50} actually said"`, `"quote"` with attribution nearby
   - `according_to`: Count of "according to" phrases
   - `page_citations`: Count of "page \d+" references
   - `document_reveals`: "notice this", "reading directly from", "look at this"
   - `quote_density`: Quoted text segments per 1000 words
   - Return counts dict

8. `analyze_pacing(text: str) -> Dict[str, Any]` — Analyze structural pacing:
   - `word_count`: Total words
   - `paragraph_count`: Count of double-newline separated blocks
   - `avg_paragraph_words`: Average words per paragraph
   - `question_count`: Total question marks (engagement signals)
   - `questions_per_1000_words`: Normalized question density

9. `analyze_transcript_file(file_path: Path) -> Dict[str, Any]` — Full pipeline for single file:
   - Call read_transcript, get_creator_name, all extract_* functions
   - Return consolidated analysis dict

10. `analyze_all_transcripts(transcripts_dir: Path) -> List[Dict]` — Batch analysis:
    - Discover all .txt, .srt, .vtt files recursively
    - Analyze each, collect results
    - Print progress to stderr
    - Return list of all analyses

11. CLI with argparse:
    - `--analyze-all` — Run full pipeline on transcripts/ folder, print JSON summary
    - `--analyze FILE` — Analyze single transcript, print JSON
    - `--stats` — Print creator/format distribution stats
    - Default transcripts dir: `../../transcripts` relative to script

Create `tools/youtube-analytics/technique_library.py` with database CRUD for creator_techniques table.

**Follow existing patterns from:**
- `tools/discovery/database.py` for migration pattern (PRAGMA user_version)
- `tools/youtube-analytics/playbook_synthesizer.py` for synthesizer integration pattern
- Error dict pattern: return `{'error': msg}`, never raise

**Class: TechniqueLibrary**

Constructor: `__init__(self, db_path: str = None)` — Connect to keywords.db (default: `tools/discovery/keywords.db`), call `_ensure_schema_v28()`.

**Migration: `_ensure_schema_v28(self)`**
- Check `PRAGMA user_version` — if < 28, create table + bump version
- Table `creator_techniques`:
  ```sql
  CREATE TABLE IF NOT EXISTS creator_techniques (
      id INTEGER PRIMARY KEY AUTOINCREMENT,
      technique_category TEXT NOT NULL,
      technique_name TEXT NOT NULL,
      formula TEXT,
      when_to_use TEXT,
      creator_examples TEXT,  -- JSON: [{'creator': str, 'video': str, 'text': str}]
      creator_count INTEGER DEFAULT 1,
      is_universal BOOLEAN DEFAULT 0,
      style_guide_ref TEXT,   -- e.g., 'Part 6.2 Pattern 1' if maps to existing
      created_at DATE NOT NULL,
      UNIQUE(technique_category, technique_name)
  )
  ```
- Index on `technique_category`
- Index on `(is_universal DESC, creator_count DESC)`

**CRUD methods:**

1. `add_technique(category, name, formula, when_to_use, creator_examples, is_universal=False, style_guide_ref=None) -> Dict`
   - UPSERT with ON CONFLICT UPDATE
   - Auto-calculate creator_count from unique creators in examples
   - Return `{'technique_id': int, 'action': 'inserted'|'updated'}` or `{'error': msg}`

2. `get_techniques_by_category(category: str) -> List[Dict]`
   - Return all techniques in category, ordered by creator_count DESC
   - Parse creator_examples from JSON back to list

3. `get_universal_techniques() -> List[Dict]`
   - Return all techniques where is_universal=True, ordered by creator_count DESC

4. `get_all_techniques() -> List[Dict]`
   - Return everything, grouped by category

5. `get_technique_categories() -> List[str]`
   - Return DISTINCT categories

6. `search_techniques(query: str) -> List[Dict]`
   - Search technique_name, formula, when_to_use with LIKE %query%

7. `store_analysis_results(analyses: List[Dict]) -> Dict`
   - Take output from transcript_analyzer.analyze_all_transcripts()
   - Transform extracted patterns into technique rows
   - Group by pattern type, aggregate creator examples
   - Call add_technique for each unique pattern
   - Return `{'stored': int, 'updated': int, 'errors': int}`

8. `get_stats() -> Dict`
   - Return counts: total techniques, universal count, by-category breakdown

**CLI with argparse:**
- `--store-from FILE` — Read JSON analysis results, store to DB
- `--list [CATEGORY]` — List techniques (all or by category)
- `--search QUERY` — Search techniques
- `--stats` — Show database statistics
- `--db PATH` — Override database path (default: `../../tools/discovery/keywords.db`)
