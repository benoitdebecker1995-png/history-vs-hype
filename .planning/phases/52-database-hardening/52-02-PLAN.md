<objective>
Wrap existing keywords.db and technique_library.py migrations in atomic transactions.

Purpose: Both databases have migration methods that execute DDL without transaction protection. If a migration fails mid-way (e.g., disk full, concurrent access), the database is left in a partial state with no rollback. This plan wraps each migration block in `with self._conn:` for automatic rollback on failure.

Output: database.py and technique_library.py with transaction-wrapped migrations that satisfy DB-02 and DB-03.
</objective>
