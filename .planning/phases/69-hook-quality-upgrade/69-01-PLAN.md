  <name>Task 2: Upgrade hook_scorer.py — framework detection, fulfillment check, style recommendation</name>
  <files>tools/research/hook_scorer.py</files>
  <action>
    **Part A: Replace _detect_beats() with _detect_framework()**

    Replace the 4-beat detection with 3-element Document Reveal framework detection. Point allocation within 40-point budget: anomaly=15, stakes=15, inciting_incident=10.

    Detection signals (from 69-RESEARCH.md):
    - anomaly (0-15): specific year/number in first 30 words, named document/map, hyper-specific place. Use regex for 3-4 digit numbers, named entities (capitalized pairs), document keywords (map, document, telegram, treaty, decree, clause, line, statute).
    - stakes (0-15): physical consequence language in first 112 words — "which meant", "this meant/determined/controlled", "entire", "all of", "million", "empire", "shut off", "resource", "consequence", "controlled".
    - inciting_incident (0-10): turn/pivot language within first 112 words — "but", "except", "until", "the problem was", "that's when", "here's what", "what actually", "unless".

    Result dict changes: 'beat_score' → 'framework_score', 'beats' → 'framework'. Each framework element is scored as a float (0 to max), not boolean. Full points if strongly present, partial (e.g., half) if weakly present (Claude's discretion on partial scoring heuristic — simplest: binary full/zero is acceptable).

    **Part B: Add _check_entity_echo() and _check_promise_type()**

    Entity echo: Use EntityExtractor(use_spacy=False).extract(title) to get title entities. Check if at least one entity's .text appears (case-insensitive) in first 50 words of hook. Return {passed: bool, matched_entities: list, title_entities: list}.

    Promise-type classification: Categorize title by keyword matching:
    - mechanism: "how", "why", "what made", "the reason"
    - document: "law", "treaty", "telegram", "document", "statute", "decree", "map"
    - conflict: "vs", "versus", "war", "rivalry", "divided", "split"
    - myth-bust: "myth", "truth", "real", "actually", "wasn't", "never"

    Verify hook's first 50 words match the same type:
    - mechanism: causal language ("because", "which meant", "thereby", "led to")
    - document: named document in first 50 words
    - conflict: two entities in opposition
    - myth-bust: belief language ("you've heard", "most people", "standard answer")

    When fulfillment fails, generate fix suggestion: "Title promises {type} about {entities}, but hook opens with {what_hook_does}. Try: {specific_suggestion}."

    Fulfillment result is displayed as pass/fail flags, NOT added to the 100-point total_score (per Research pitfall #2 recommendation — keeps score meaning stable).

    **Part C: Add _load_pattern_library() and style recommendation**

    Parse .claude/REFERENCE/HOOK-PATTERN-LIBRARY.md at runtime. Split by "## Pattern:" delimiter. Count examples per pattern. Extract topic types and first-sentence examples.

    Library path: resolve relative to the repo root. Use pathlib — try Path(__file__).resolve().parents[2] / '.claude/REFERENCE/HOOK-PATTERN-LIBRARY.md'. If file missing, return empty dict (graceful degradation).

    Topic-to-style mapping:
    - territorial → cold_fact
    - ideological → myth_contradiction
    - political_fact_check → specificity_bomb
    - general → None (no recommendation)

    Score impact: +5 when matching recommended style AND library has 7+ examples for that pattern (HIGH confidence). -5 when mismatching at HIGH confidence. Advisory-only (no score change) when <5 examples (LOW confidence). Style score is additive to total but total capped at min(100, max(0, total)).

    Style recommendation result: {recommended: str, confidence: 'high'|'low'|'none', examples: list[str], score_modifier: int}.

    **Part D: Update score_hook() signature and format functions**

    New signature: score_hook(text, label='', title=None, topic_type=None). All new params default None.

    When title is not None: run fulfillment check, add 'fulfillment' key to result.
    When topic_type is not None: run style recommendation, add 'style_recommendation' key to result. Apply score modifier to total.
    When both None: behave exactly as before (backward compat) except framework replaces beats.

    Update format_hook_ranking() to show Framework column instead of Beats. Show fulfillment pass/fail if present. Show style recommendation if present.

    Update rank_hooks() to pass title and topic_type through if provided.

    **Part E: Add detect_topic_from_script()**

    Add TOPIC_KEYWORDS dict and detect_topic_from_script(text) function that keyword-matches first 2000 chars to determine topic type. Returns 'territorial', 'ideological', 'political_fact_check', or 'general'. Same pattern as title_scorer.py detect_pattern() but extended to script content.

    This function is exported for use by the /script --hooks command (Plan 02).
