  <name>Task 1: Create topic_strategy.py</name>
  <files>tools/youtube-analytics/topic_strategy.py</files>
  <action>
Create `tools/youtube-analytics/topic_strategy.py` with the following functions:

**1. `generate_topic_strategy()`**
- Query video_performance table via KeywordDB for all videos with performance data
- Group by topic_type (use TAG_VOCABULARY categories from performance.py: territorial, ideological, colonial, legal, general, political, educational)
- For each topic type, calculate:
  - avg_retention (from retention data if available, or from video_performance.avg_retention_pct)
  - avg_conversion_rate (subscribers gained / views * 100)
  - video_count
  - best_video (highest conversion in that category)
  - worst_video (lowest conversion)
  - common_observations (from lessons_learned JSON, aggregated)
- Sort by avg_conversion_rate descending
- Generate concrete_next_steps based on patterns:
  - If best topic has >1.5x the average conversion: "Prioritize {topic} -- {rate}% conversion vs {avg}% channel average"
  - If a topic has 0 videos: skip (don't recommend what hasn't been tried)
  - If worst topic has <0.5x average: "Reconsider {topic} approach or apply {best_topic} patterns"
  - Cross-topic recommendations: "Apply {best_topic}'s successful patterns (e.g., {top_observation}) to {underperformer}"
- Flag low-confidence insights: add `confidence: 'low'` when video_count < 3, `'medium'` for 3-5, `'high'` for 6+
- Returns:
  ```python
  {
      'topic_stats': [
          {
              'topic': 'territorial',
              'avg_retention': 0.34,
              'avg_conversion': 0.42,
              'video_count': 5,
              'confidence': 'high',
              'best_video': {'title': '...', 'conversion': 0.8},
              'worst_video': {'title': '...', 'conversion': 0.1},
              'common_observations': ['Strong retention in treaty sections', ...]
          },
          ...
      ],
      'best_performing': [...],  # Top 2 topic types
      'worst_performing': [...],  # Bottom 2 topic types
      'concrete_next_steps': [
          'Prioritize territorial disputes -- 0.42% conversion vs 0.28% channel average',
          'Apply zero-impact pattern from territorial videos to ideological topics',
          ...
      ],
      'channel_average_conversion': float,
      'total_videos': int,
      'generated_at': str  # ISO timestamp
  }
  ```

**2. `format_strategy_terminal(strategy)`**
- Compact ASCII table for terminal display
- Topic | Videos | Avg Retention | Conversion | Confidence
- Sorted by conversion descending
- Next steps as numbered list below table
- ASCII-only for Windows cp1252 compatibility

**3. `format_strategy_markdown(strategy)`**
- Full markdown report suitable for saving
- Performance comparison table
- Best/worst analysis with video examples
- Concrete recommendations section
- Confidence flags on all insights

**Error handling:** Return `{'error': msg}` if database unavailable or empty. Error dict pattern per project convention.

**Imports:** Use KeywordDB from database.py (add sys.path for discovery), statistics.mean for averages, json for JSON parsing. No new external dependencies.

**CLI:** Add `if __name__ == '__main__'` with:
- `python topic_strategy.py` -- terminal format
- `python topic_strategy.py --markdown` -- markdown format
- `python topic_strategy.py --save` -- save to channel-data/TOPIC-STRATEGY.md

Add the following new function to feedback_queries.py (append after existing functions, before formatting section):

**`get_pre_script_insights(topic_type, limit=5)`**
- Combines multiple insight sources into a single pre-script briefing:
  1. Topic-specific feedback (from existing get_insights_for_topic with command='script')
  2. Topic strategy summary (from new topic_strategy.generate_topic_strategy)
  3. Section-level retention lessons (query videos of same topic_type, extract drop point observations)
- Returns structured dict:
  ```python
  {
      'topic_type': str,
      'video_count': int,          # How many past videos inform these insights
      'confidence': 'high' | 'medium' | 'low',
      'retention_lessons': [
          {
              'lesson': 'Lost 12% viewers at treaty text section',
              'recommendation': 'Show treaty on screen, don\'t read aloud',
              'source_video': 'Belize (2025-01-15)',
              'drop_magnitude': 0.12
          },
          ...
      ],
      'topic_strategy_summary': str,  # One-line from topic strategy ("territorial: 0.42% conversion, highest performing")
      'pattern_suggestions': [
          'Territorial videos benefit from zero-impact moments (Part 6.4 Pattern 2)',
          ...
      ],
      'display_text': str  # Pre-formatted text ready to display (replaces get_insights_preamble for /script)
  }
  ```

**`format_pre_script_display(insights)`**
- Formats the pre-script insights as a compact, readable block for display before /script generation
- Format:
  ```
  --- Pre-Script Intelligence ({topic_type}, {confidence} confidence) ---

  Topic Performance: {topic_strategy_summary}

  Retention Lessons ({n} past videos):
  - [Video Title]: Lost X% at {section} -- {recommendation}
  - ...

  Suggested Patterns:
  - {pattern_suggestion_1}
  - {pattern_suggestion_2}
  ---
  ```
- Returns empty string if no insights available

**Integration with topic_strategy:**
- Add optional import of topic_strategy at top of file with try/except and TOPIC_STRATEGY_AVAILABLE flag
- get_pre_script_insights calls generate_topic_strategy() if available, extracts summary for the requested topic
- Falls back gracefully if topic_strategy not available (returns insights without strategy summary)

**Maintain backward compatibility:**
- Existing get_insights_preamble() remains unchanged (other commands may use it)
- New get_pre_script_insights() is the enhanced replacement specifically for /script context
- /script command (updated in Plan 35-03) will call get_pre_script_insights instead of get_insights_preamble
