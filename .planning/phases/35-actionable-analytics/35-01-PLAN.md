Create `tools/youtube-analytics/retention_mapper.py` with the following functions:

**1. `map_retention_to_sections(drop_off_points, sections, wpm=150)`**
- Input: list of drop-off dicts from retention.py (`{position, retention_before, retention_after, drop, timestamp_hint}`), list of Section objects from parser.py
- Does NOT call YouTube API or parse scripts itself -- receives pre-fetched data
- Calculates cumulative word percentages for each section: `start_pct = running_total / total_words`, `end_pct = (running_total + section.word_count) / total_words`
- For each drop, finds which section it falls in by matching `drop.position` against section boundaries
- Returns list of mapped drop dicts:
  ```python
  {
      'section_heading': str,        # H2 heading text
      'section_type': str,           # intro/body/conclusion
      'drop_position': float,        # 0.0-1.0
      'drop_magnitude': float,       # positive, e.g. 0.08 = 8%
      'retention_before': float,
      'retention_after': float,
      'word_range': (int, int),      # (start_word, end_word)
      'estimated_timestamp': str,    # e.g. "1:30-3:15" based on WPM
      'section_content_preview': str, # First 100 chars of section
      'position_in_section': float   # 0.0-1.0 where within section the drop occurred
  }
  ```
- Handle edge cases: empty drops list (return []), empty sections list (return []), drops outside any section boundary (skip)

**2. `estimate_section_timestamps(sections, wpm=150)`**
- Input: list of Section objects
- Returns list of dicts: `{heading, start_time_str, end_time_str, duration_seconds, word_count}`
- Format timestamps as "M:SS" (e.g., "1:30", "5:45")

**3. `format_mapped_drops_table(mapped_drops)`**
- Returns formatted markdown table string of mapped drops, sorted by magnitude (biggest first)
- Columns: Section | Drop | Retention | Est. Time | Severity
- Severity: HIGH (>10%), MEDIUM (5-10%), LOW (<5%)

**Error handling:** Return empty list on any error (never raise). Use error dict pattern only if returning dict format.

**Imports:** Use ScriptParser and Section from parser.py via relative path. Add `sys.path.insert(0, str(Path(__file__).parent.parent / 'production'))` at top.

**TDD approach:**
- Write tests first for: single section mapping, multi-section mapping, edge cases (empty inputs, drop at boundary), timestamp estimation
- Tests use mock Section objects and mock drop data (no API calls)
- Test file: `tools/youtube-analytics/test_retention_mapper.py`

Create `tools/youtube-analytics/section_diagnostics.py` with the following functions:

**1. `load_voice_patterns()`**
- Returns a dict of the 22 voice patterns from STYLE-GUIDE.md Part 6, organized by category
- Hardcode the pattern data (not parse markdown at runtime -- the patterns are stable and documented)
- Structure:
  ```python
  {
      'openings': {
          'visual_contrast_hook': {
              'name': 'Visual Contrast Hook',
              'formula': '[Show visual A]. [Show visual B]. [State the tension].',
              'when': 'Territorial disputes with map changes',
              'style_guide_ref': 'Part 6.1 Pattern 1'
          },
          # ... 4 more opening patterns
      },
      'transitions': {
          'causal_chain': {
              'name': 'Kraut-Style Causal Chain',
              'formula': '[Event]. Consequently, [result]. Which meant that [implication].',
              'when': 'Explaining causation, not just sequence',
              'style_guide_ref': 'Part 6.2 Pattern 1'
          },
          # ... 7 more transition patterns
      },
      'evidence': { ... },  # 5 patterns
      'rhythm': { ... },    # 4 patterns
  }
  ```

**2. `diagnose_section_drop(section_text, section_heading, section_type, drop_magnitude, position_in_section=0.5)`**
- Analyzes WHY viewers dropped in this section
- Checks for anti-patterns by analyzing section content:
  - Abstract opening (starts with "The concept", "To understand", "The idea", "In order to")
  - Missing causal chains (no "consequently", "thereby", "which meant that", "as a result")
  - No evidence introduction (no quotes, no "according to", no page numbers)
  - Long paragraphs without breaks (>150 words without pattern interrupt markers)
  - Missing modern relevance (no "today", "2024", "2025", "2026", "currently", "modern")
  - Section-type specific: intro without hook detection, body without transitions, conclusion without synthesis
- Recommends specific voice patterns from Part 6:
  - Missing causal chain -> recommend "Kraut-Style Causal Chain" (Part 6.2 Pattern 1)
  - No evidence -> recommend "Setup -> Quote -> Implication" (Part 6.3 Pattern 1)
  - Abstract opening -> recommend "Visual Contrast Hook" or "Current Event Hook" (Part 6.1)
  - Pacing issues -> recommend "Long Setup + Short Punch" (Part 6.4 Pattern 1) or "Question -> Zero Answer" (Part 6.4 Pattern 2)
- Returns:
  ```python
  {
      'root_causes': ['Abstract opening - no concrete anchor', ...],
      'recommendations': [
          {
              'fix': 'Start with concrete date/place/document',
              'pattern': 'Visual Contrast Hook',
              'pattern_ref': 'STYLE-GUIDE.md Part 6.1 Pattern 1',
              'insertion_hint': 'Replace first sentence with: [Show map A]. [Show map B]. [State tension].'
          },
          ...
      ],
      'severity': 'HIGH' | 'MEDIUM' | 'LOW',
      'confidence': 'high' | 'medium' | 'low'  # Based on drop magnitude + number of anti-patterns found
  }
  ```

**3. `diagnose_all_drops(mapped_drops, sections)`**
- Wraps diagnose_section_drop for all mapped drops
- Matches each mapped drop back to its section content from sections list
- Returns list of diagnosis dicts, one per drop, sorted by severity then magnitude
- Adds 'section_heading' and 'drop_magnitude' to each diagnosis

**4. `format_diagnostics_markdown(diagnoses)`**
- Formats all diagnoses as markdown report
- Groups by severity (HIGH first, then MEDIUM, then LOW)
- Each diagnosis shows: section name, drop %, root causes, recommended patterns with references
- Includes confidence flag when n < 3 past videos ("Low confidence - limited historical data")

**Error handling:** Return `{'root_causes': [], 'recommendations': [], 'severity': 'LOW', 'confidence': 'low'}` on any error.
