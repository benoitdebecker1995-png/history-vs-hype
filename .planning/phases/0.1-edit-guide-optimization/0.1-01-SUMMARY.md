# Phase 0.1 Plan 01: Edit Guide Optimization Summary

**One-liner:** Restructured edit-guide.md with clean section flow 1-6, concrete cut detection heuristics by priority level, and runtime estimation guidance with delta interpretation table.

## What Was Delivered

### 1. Clean Section Structure
- Removed duplicate section 4 ("REQUIRED SECTIONS") and section 5 ("QUALITY STANDARDS")
- Consolidated quality standards into a compact block under shot-by-shot section
- Sections now flow cleanly: 1. Header, 2. Cut Candidates, 3. Pacing Analysis, 4. Editing Philosophy, 5. SRT Corrections, 6. Shot-by-Shot

### 2. Cut Detection Heuristics
Added concrete detection criteria for each priority level:

**Priority 1 (Easy Cuts):** Repeated explanations, filler phrases, unnecessary context, tangents, long pauses, hedging language

**Priority 2 (Moderate Cuts):** Extra examples beyond first two, summarizable context, shortable quotes, secondary evidence, over-explained transitions

**Priority 3 (Hard Cuts):** Author "darlings," competing secondary arguments, evidence stacking, tangents for separate videos

**DO NOT CUT:** Opening hook, thesis statements, primary source evidence, pattern interrupts, payoff moments, call to action

### 3. Runtime Estimation Guidance
- **Formula:** Count script words, divide by 150 (channel speaking pace)
- **Delta interpretation table:**
  - +3 min or more: Needs trimming (Priority 1 then 2)
  - +1-3 min: Light trimming (Priority 1 sufficient)
  - On target: Focus on pacing
  - -1 min or more: Too short, may be rushed
