<objective>
Add academic source hint generation to production constraint evaluation.

Purpose: Give user a quick sense of academic source availability BEFORE investing in NotebookLM research. Not a substitute for actual research - just a pre-screening signal.

Approach: Generate search query templates that user can manually run in Google Scholar, JSTOR, WorldCat. No API calls (no official Scholar API exists, SERP APIs expensive). Store hint results when user confirms.

Output:
- get_source_hints() returns search queries + expected source types
- evaluate_production_constraints() includes source_hints in output
- CLI shows source hints alongside document score and animation check
</objective>

**get_source_hints(title, description=''):**
- Generate search query templates based on topic keywords
- Return dict with:
  - queries: list of search strings ready to copy-paste
  - expected_types: ['monograph', 'journal', 'primary'] based on topic

  - `"Sykes-Picot" primary sources archive`
  - `"Dark Ages myth" peer-reviewed`
