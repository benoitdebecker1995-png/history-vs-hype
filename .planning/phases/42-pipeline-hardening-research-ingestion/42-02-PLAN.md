Create `tools/research/nlm_ingest.py` with these components:

**1. NLMParser class — Parse raw NotebookLM output into structured chunks**

`parse(text: str) -> dict` that handles both formats:
- **Structured format:** Bullet points with inline citations like `According to Smith (2020, p.45), the treaty established...` or `[Source 1, p.23]` references
- **Freeform format:** Paragraphs mixing claims with discussion

Parsing strategy (Claude's discretion per CONTEXT.md — use hybrid approach):
- First pass: regex to find citation patterns: `(Author, Year, p.XX)`, `[Source N, p.XX]`, `page XX`, `pp. XX-YY`
- Second pass: split on sentence boundaries near citation markers
- Each chunk = one claim-candidate with its citation text preserved
- Return: `{'chunks': [{'text': str, 'citation': str, 'source_ref': str, 'line_number': int}], 'parse_stats': {'total_lines': int, 'chunks_found': int, 'citation_coverage': float}}`

**2. ClaimExtractor class — Categorize and structure extracted claims**

`extract_claims(chunks: list) -> dict` that:
- Categorizes each chunk by type using keyword heuristics:
  - `statistic`: contains numbers, percentages, "GDP", "population", "rate"
  - `quote`: contains quotation marks or "said", "wrote", "argued"
  - `event`: contains dates (YYYY, "century", "era"), "war", "treaty", "battle"
  - `definition`: contains "defined as", "means", "refers to", "is called"
  - `claim`: everything else (general historical claims)
- Assigns confidence: `high` (has page number + author), `medium` (has author or page), `low` (no specific citation)
- Return: `{'claims': [{'id': int, 'text': str, 'type': str, 'citation': str, 'confidence': str}], 'by_type': {'statistic': int, 'quote': int, ...}}`

**3. ReviewGenerator class — Create markdown checklist for user review**

`generate_review_file(claims: list, output_path: str) -> dict` that:
- Creates a markdown file at `output_path` with structure:
  ```markdown
  # NLM Claim Review - [timestamp]

  **Instructions:** Review each claim below. For each claim:
  - [x] = APPROVE (will be written to VERIFIED-RESEARCH.md)
  - [ ] = REJECT (will be skipped)
  - Edit the text/citation directly to correct before approving

  **Target file:** [path to 01-VERIFIED-RESEARCH.md]

  ---

  ## Statistics (3 claims)

  - [ ] **[1]** GDP growth averaged 4.1% annually from 1928-1970
    - Citation: Robert Allen, *Farm to Factory*, p. 142
    - Confidence: high

  - [ ] **[2]** Population increased by 35% between 1940-1960
    - Citation: Source 3, p. 89
    - Confidence: medium

  ## Quotes (2 claims)
  ...
  ```
- Group claims by type (statistics first, then quotes, events, definitions, general claims)
- Include claim ID in bold brackets for tracking
- Return: `{'path': str, 'claim_count': int}` or `{'error': str}`

**4. ReviewReader class — Parse the edited review file to get approvals**

`read_approvals(review_path: str) -> dict` that:
- Reads the markdown file back
- Finds checked items: `- [x]` = approved, `- [ ]` = rejected
- Extracts the (potentially edited) text and citation for each approved claim
- Return: `{'approved': [{'id': int, 'text': str, 'citation': str, 'type': str}], 'rejected_count': int, 'approved_count': int}`

**5. VerifiedResearchWriter class — Write approved claims to VERIFIED-RESEARCH.md**

`write_claims(claims: list, verified_research_path: str) -> dict` that:
- Validates the file exists (error if not — user must set up project first via /research --new)
- Reads existing content to find insertion points by section
- Maps claim types to VERIFIED-RESEARCH.md sections:
  - `statistic` → "## KEY STATISTICS" or "## ECONOMIC DATA" section (look for closest match)
  - `quote` → "## KEY QUOTES" section
  - `event` → "## TIMELINE" or "## KEY EVENTS" section
  - `definition` → "## DEFINITIONS" section
  - `claim` → "## CLAIMS TO VERIFY" section (default catch-all)
- For each claim, appends in VERIFIED-RESEARCH format:
  ```markdown
  ### [Claim text summary - first 60 chars]
  - **Status:** ✅ VERIFIED (via NLM ingestion)
  - **Claim:** [full claim text]
  - **Source:** [citation]
  - **Ingested:** [date]
  ```
- If no matching section found, appends to end under "## INGESTED CLAIMS (Unsorted)"
- Return: `{'written': int, 'sections_updated': list, 'path': str}` or `{'error': str}`

**6. CLI function — Orchestrate the flow**

`ingest(input_text: str = None, input_file: str = None, project_path: str = None) -> dict` that:
1. Read input (from text or file)
2. Parse with NLMParser
3. Extract claims with ClaimExtractor
4. Generate review file in project's `_research/` folder as `_NLM-REVIEW-[timestamp].md`
5. Return the review file path and stats for the slash command to display

`apply_review(review_path: str, verified_research_path: str) -> dict` that:
1. Read approvals from review file
2. Write approved claims to VERIFIED-RESEARCH.md
3. Return write stats

**Error handling:** All functions return `{'error': str}` on failure, never raise. Validate project_path exists, VERIFIED-RESEARCH.md exists, review file exists.
