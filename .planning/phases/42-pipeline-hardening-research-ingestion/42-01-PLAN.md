Create `tools/translation/env_loader.py` with:
- `load_api_key()` function that:
  1. Reads `.env` file from project root (`G:/History vs Hype/.env`) using pure Python (no dotenv dependency — parse `KEY=value` lines, skip comments `#`, strip quotes)
  2. Falls back to `os.environ.get('ANTHROPIC_API_KEY')` if .env file doesn't exist or key not found
  3. Returns `{'key': str}` on success or `{'error': str}` on failure
  4. On missing key, returns actionable error: `"Missing ANTHROPIC_API_KEY.\n\nTo fix:\n1. Get your API key from https://console.anthropic.com/settings/keys\n2. Add it to .env file:\n   echo ANTHROPIC_API_KEY=sk-ant-... >> \"G:/History vs Hype/.env\"\n\nOr set it as environment variable:\n   export ANTHROPIC_API_KEY=sk-ant-..."`
- `ensure_env_example()` function that creates `.env.example` at project root if it doesn't exist, containing:
  ```
  # Anthropic API Key (required for translation pipeline)
  # Get yours at: https://console.anthropic.com/settings/keys
  # ANTHROPIC_API_KEY=sk-ant-...
  ```
- `wrap_api_error(exception)` function that converts common API exceptions into actionable messages:
  - `anthropic.AuthenticationError` → "Invalid API key. Check your key at https://console.anthropic.com/settings/keys"
  - `anthropic.RateLimitError` → "Rate limit hit. Wait 60 seconds and retry, or check your plan at https://console.anthropic.com/settings/plans"
  - `anthropic.APIConnectionError` / connection errors → "Network error: Could not reach Anthropic API. Check your internet connection and try again."
  - All others → "API error: {str(e)}. If this persists, check https://status.anthropic.com"

Update **all 5 modules** that currently use `os.environ.get('ANTHROPIC_API_KEY')`:
- `translator.py`, `cross_checker.py`, `legal_annotator.py`, `surprise_detector.py`, `verification.py`
- In each `__init__` method: replace `os.environ.get('ANTHROPIC_API_KEY')` with `from env_loader import load_api_key; result = load_api_key(); if 'error' in result: self.error = result['error']; return`
- In each API call (translate_clause, check_clause, etc.): wrap the `self.client.messages.create(...)` call in try/except that uses `wrap_api_error()` for the error message instead of raw `str(e)`
- Call `ensure_env_example()` from env_loader on first import (module-level)

Create `.env.example` at project root with the commented template.

**IMPORTANT:** Do NOT use python-dotenv package. Parse .env manually (it's 15 lines of code). The project minimizes external dependencies.

Create `tools/translation/smoke_test.py` that validates the full pipeline end-to-end:

**Test document:** Embed a short 3-clause French legal snippet (hardcoded, ~100 words) — a fabricated but realistic legal article with preamble, two articles, and a closing clause.

**Smoke test flow (5 steps):**
1. **Credential check:** Call `load_api_key()` — if error, print actionable message and exit with status 1
2. **Structure detection:** Run `StructureDetector().detect_structure(test_doc)` — verify returns sections (not error), verify section_count >= 2
3. **Translation:** Run `Translator().translate_document(sections, 'french', test_doc)` — verify returns translated sections (not error), verify each section has 'translation' key with non-empty string
4. **Formatting:** Run `Formatter().format_paired(translated_sections)` — verify output is non-empty string containing "##"
5. **Pipeline summary:** Print pass/fail for each step with timing

**Output format:**
```
=== Translation Pipeline Smoke Test ===

[1/5] Credential check.......... PASS (API key loaded from .env)
[2/5] Structure detection....... PASS (3 sections detected)
[3/5] Translation............... PASS (3 clauses translated, 4.2s)
[4/5] Formatting................ PASS (markdown output generated)
[5/5] Pipeline integrity........ PASS (all steps connected)

Result: ALL PASSED (5/5)
Total time: 5.1s
```

On any failure, print the step that failed with the error message and suggest a fix. Exit with status 0 on all pass, status 1 on any fail.

**CLI integration:** Add a `smoketest` subcommand to `cli.py`:
- `python tools/translation/cli.py smoketest`
- Routes to `smoke_test.run_smoke_test()` function
- No arguments needed

**IMPORTANT:** The smoke test makes real API calls (1-3 translation calls for the short test document). This is intentional — it validates the actual API connection, not mocks.
