Create `tools/youtube-analytics/feedback_parser.py` — the markdown extraction module.

This module parses POST-PUBLISH-ANALYSIS markdown files into structured dicts. Use best-effort regex matching per user decision (not full markdown AST parsing).

**Functions to implement:**

1. `parse_analysis_file(filepath: str) -> dict` — Main entry point. Opens file, calls extractors, returns complete parsed dict or `{'error': msg}`. Must handle both project-folder files (have header with Video ID/Analyzed/Saved to) and fallback files (in channel-data/analyses/).

2. `extract_video_id(content: str) -> Optional[str]` — Extract from `**Video ID:** XXXX` pattern. Fallback: extract from filename pattern `POST-PUBLISH-ANALYSIS-{VIDEO_ID}.md`.

3. `extract_metrics(content: str) -> dict` — Extract numeric performance data:
   - `avg_retention`: float or None — from `**Average retention:** XX.X%`
   - `final_retention`: float or None — from `**Final retention:** XX.X%`
   - `ctr`: float or None — from `**CTR:** XX.X%` (skip if "Not available via API")
   - `views`: int or None — from Performance table, "Views" row, "This Video" column
   - `subscribers_gained`: int or None — from Performance table, "Subscribers" row (strip +)
   - `impressions`: int or None — from `**Impressions:** X,XXX`

4. `extract_lessons(content: str) -> dict` — Extract qualitative insights:
   - `observations`: list[str] — from `### Observations` section, each `- ` bullet
   - `actionable`: list[str] — from `### Actionable Items` section, each `- [ ] ` bullet (strip checkbox)

5. `extract_drop_points(content: str) -> list[dict]` — Extract from Drop-off Points table:
   - Each row: `{'position_pct': int, 'viewers_lost_pct': float, 'location': str}`
   - Parse from `| XX% | XX.X% dropped | location |` pattern

6. `extract_discovery_diagnosis(content: str) -> Optional[dict]` — Extract from Discovery Diagnostics section:
   - `primary_issue`: str — from `**Primary Issue:** XXX`
   - `severity`: str — from `(Severity: XXX)`
   - `summary`: str — from `**Diagnosis:** XXX`

7. `find_analysis_files(project_root: Path) -> list[Path]` — Scan for all POST-PUBLISH-ANALYSIS files:
   - Search `video-projects/_IN_PRODUCTION/*/POST-PUBLISH-ANALYSIS.md`
   - Search `video-projects/_READY_TO_FILM/*/POST-PUBLISH-ANALYSIS.md`
   - Search `video-projects/_ARCHIVED/*/POST-PUBLISH-ANALYSIS.md`
   - Search `channel-data/analyses/POST-PUBLISH-ANALYSIS-*.md`
   - Return list of Path objects

8. `backfill_all(project_root: Path, force: bool = False) -> dict` — Process all files:
   - Call find_analysis_files() to get list
   - For each file, call parse_analysis_file()
   - Call db.store_video_feedback() for each parsed result
   - Skip files where video_id already has feedback (unless force=True)
   - Print progress: `[N/M] Parsing: filename... OK/SKIP/ERROR`
   - Return `{'processed': N, 'skipped': N, 'errors': N, 'details': [...]}`

**Regex patterns to handle (from examining actual files):**
- `**Average retention:** 39.4%` (multiply by 1 since already percentage)
- `**Final retention:** 27.9%`
- `**CTR:** Not available via API` (return None)
- `**CTR:** 4.5%` (return float)
- `| Views | 5,088 | 628 | +710% |` (extract "This Video" column, strip commas)
- `| Subscribers | +51 | +1 | +5000% |` (extract number, strip +)
- `- Strong retention (39.4%) - hook and pacing working well` (observation bullets)
- `- [ ] This topic type works well` (actionable items, strip `[ ] `)
- `| 3% | 7.0% dropped | intro |` (drop-off rows)
- `**Video ID:** XbGl1Kcspt4` (video ID)
- `**Analyzed:** 2026-02-03T...` (analysis date)

**Error handling:** Every function returns its data or None/empty-list on failure. parse_analysis_file wraps all in try/except and returns `{'error': msg}` on failure. Never raise exceptions to caller.

**Imports:** Only stdlib: `re`, `pathlib`, `datetime`, `json`, `sys`. Add `sys.path.insert(0, ...)` for KeywordDB import in backfill_all.

**Part A: Add feedback CRUD methods to KeywordDB in database.py**

Add these methods to the KeywordDB class (after existing variant methods, before close()):

1. `store_video_feedback(self, video_id: str, feedback_data: dict) -> dict`
   - Updates video_performance row with parsed feedback
   - Sets `retention_drop_point` = feedback_data.get('biggest_drop_position') (int, percentage position of biggest drop)
   - Sets `discovery_issues` = json.dumps(feedback_data.get('discovery', {})) if present, else None
   - Sets `lessons_learned` = json.dumps({'observations': feedback_data.get('observations', []), 'actionable': feedback_data.get('actionable', [])})
   - Uses UPDATE WHERE video_id = ?
   - Returns `{'status': 'updated', 'video_id': video_id}` on success
   - Returns `{'status': 'no_match', 'video_id': video_id}` if rowcount == 0 (video not in performance table)
   - Returns `{'error': msg}` on exception
   - Follow error dict pattern from Phase 16+

2. `get_video_feedback(self, video_id: str) -> dict`
   - Queries video_performance for feedback columns + basic info
   - SELECT video_id, title, topic_type, conversion_rate, retention_drop_point, discovery_issues, lessons_learned FROM video_performance WHERE video_id = ?
   - Parse JSON columns with null-safety: `json.loads(val) if val else None`
   - Returns `{'video_id': ..., 'title': ..., 'topic_type': ..., 'conversion_rate': ..., 'drop_point': ..., 'discovery': ..., 'lessons': ...}`
   - Returns `{'error': 'not_found'}` if no row
   - Returns `{'error': msg}` on exception

3. `get_feedback_by_topic(self, topic_type: str, limit: int = 10) -> dict`
   - Queries all videos in topic_type that have lessons_learned
   - SELECT video_id, title, topic_type, conversion_rate, retention_drop_point, lessons_learned FROM video_performance WHERE topic_type = ? AND lessons_learned IS NOT NULL ORDER BY conversion_rate DESC LIMIT ?
   - Returns `{'videos': [...], 'count': N, 'topic': topic_type}`
   - Each video dict includes parsed lessons_learned (json.loads)

4. `has_feedback(self, video_id: str) -> bool`
   - Quick check: SELECT lessons_learned FROM video_performance WHERE video_id = ? AND lessons_learned IS NOT NULL
   - Returns True/False

**IMPORTANT:** Import json at top of database.py if not already imported. Add `import json` near existing imports.
