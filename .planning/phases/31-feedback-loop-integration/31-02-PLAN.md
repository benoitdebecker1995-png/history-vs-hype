Create `tools/youtube-analytics/feedback_queries.py` — the query and pattern extraction module.

**Imports:** stdlib only (sys, json, pathlib, statistics, typing). Import KeywordDB via sys.path pattern. Import pattern_extractor functions optionally with try/except.

**Functions to implement:**

1. `get_insights_for_topic(topic_type: str, command: str = 'script', limit: int = 5) -> dict`
   - Retrieves category-specific insights from videos matching topic_type
   - Uses KeywordDB.get_feedback_by_topic(topic_type, limit)
   - Parses lessons_learned JSON from each video
   - Filters insights by command type (per user decision):
     - `command='script'`: Return content and pacing insights (observations about retention, section structure, hooks)
     - `command='prep'`: Return production insights (observations about B-roll, edit pacing, visual evidence)
     - `command='publish'`: Return CTR and title insights (observations about CTR, thumbnails, metadata)
   - Filter logic: Keyword matching on observation text. For 'script': look for 'retention', 'pacing', 'hook', 'section', 'structure', 'opening', 'engagement'. For 'prep': look for 'b-roll', 'visual', 'edit', 'production', 'thumbnail'. For 'publish': look for 'ctr', 'title', 'thumbnail', 'impressions', 'click', 'metadata'. If no keyword-filtered insights found, return all (fallback).
   - Returns `{'insights': [...], 'count': N, 'topic': topic_type, 'command': command}`
   - Each insight: `{'video_id': str, 'title': str, 'observation': str, 'conversion_rate': float}`

2. `get_universal_insights(limit: int = 2) -> dict`
   - Retrieves insights that apply to ALL videos regardless of topic
   - Query all videos with feedback, sorted by conversion_rate DESC
   - Look for universal patterns: observations containing 'pacing', 'engagement', 'conversion', 'average'
   - Returns `{'insights': [...], 'count': N}`

3. `get_insights_preamble(topic_type: str, command: str = 'script') -> str`
   - Combines get_insights_for_topic (3-4 items) + get_universal_insights (1-2 items)
   - Formats as readable preamble text:
     ```
     --- Past Performance Insights ---

     From similar videos ({topic_type}):
     - [Video Title]: observation (conversion: X.XX%)
     - [Video Title]: observation (conversion: X.XX%)

     Universal patterns:
     - observation

     ---
     ```
   - Returns empty string if no insights available
   - This is what /script, /prep, /publish will call

4. `extract_success_patterns(threshold_type: str = 'adaptive') -> dict`
   - Per FEED-03: Extract patterns from high-performing videos
   - Adaptive threshold (per user discretion): above-average conversion per topic_type
   - Fallback: above channel-average if topic has <3 videos
   - Uses `statistics.mean()` for average calculation with try/except for small samples
   - For each high-performing video, extract:
     - Content attributes: topic_type, angles (from JSON), lessons_learned observations
     - Production attributes: retention (from drop_point — low drop = good), discovery diagnosis
   - Aggregate: find common observations, dominant topics, dominant angles
   - Returns `{'patterns': [...], 'video_count': N, 'threshold': float, 'method': str}`
   - Each pattern: `{'type': 'success', 'attribute': str, 'value': str, 'frequency': int, 'videos': [titles]}`

5. `extract_failure_patterns(threshold_type: str = 'adaptive') -> dict`
   - Per FEED-04: Extract patterns from low-performing videos
   - Adaptive threshold: below-average conversion per topic_type
   - Same structure as extract_success_patterns but inverted
   - Look for: low retention observations, high drop-off points, discovery issues
   - Returns `{'patterns': [...], 'video_count': N, 'threshold': float, 'method': str}`

6. `generate_patterns_report() -> dict`
   - Combines success + failure patterns into comprehensive report
   - Also calls pattern_extractor.extract_winning_patterns() if available (import with try/except)
   - Returns `{'success': success_patterns, 'failure': failure_patterns, 'winning': winning_patterns_or_None, 'generated_at': timestamp}`

7. `format_patterns_markdown(report: dict) -> str`
   - Formats patterns report as markdown for saving to PATTERNS.md
   - Sections: Success Patterns, Failure Patterns, Winning Patterns (from pattern_extractor), Recommendations
   - Include both content attributes and production attributes per user decision

8. `format_patterns_terminal(report: dict) -> str`
   - Compact terminal display of patterns
   - Success patterns with video counts, failure patterns with warnings
   - ASCII formatting (no unicode that breaks Windows cp1252)

9. `format_query_terminal(result: dict) -> str`
   - Format individual video or topic query results for terminal
   - Table-like layout with aligned columns
   - Truncate long observation text to 80 chars

10. `format_query_markdown(result: dict) -> str`
    - Format query results as full markdown report
    - Include all observation text, video links, conversion rates

**Adaptive threshold implementation:**
```python
from statistics import mean

def _calculate_threshold(videos, topic_type=None):
    """Calculate performance threshold adaptively."""
    if topic_type:
        topic_videos = [v for v in videos if v.get('topic_type') == topic_type]
        if len(topic_videos) >= 3:
            rates = [v.get('conversion_rate', 0) or 0 for v in topic_videos]
            return mean(rates), 'topic_average'
    # Fallback to channel average
    rates = [v.get('conversion_rate', 0) or 0 for v in videos if v.get('conversion_rate')]
    if rates:
        return mean(rates), 'channel_average'
    return 0, 'default'
```

Create `tools/youtube-analytics/feedback.py` — the CLI entry point per user decision: `python feedback.py query --topic territorial --metric retention`.

**Use argparse with subcommands** (following benchmarks.py pattern).

**Subcommands:**

1. `backfill` — Import all existing analysis files
   - `python feedback.py backfill` — parse all, skip already-stored
   - `python feedback.py backfill --force` — re-parse all, overwrite existing
   - Calls feedback_parser.backfill_all()
   - Output: progress per file, final summary

2. `query` — Query insights
   - `python feedback.py query --topic territorial` — topic insights
   - `python feedback.py query --topic territorial --metric retention` — filter by metric
   - `python feedback.py query --video VIDEO_ID` — specific video feedback
   - `python feedback.py query --topic territorial --markdown` — markdown output
   - `python feedback.py query --video VIDEO_ID --markdown` — markdown output
   - Calls feedback_queries.get_insights_for_topic() or get_video_feedback()
   - Default: terminal table format. With --markdown: full report.

3. `patterns` — Generate patterns report
   - `python feedback.py patterns` — terminal display
   - `python feedback.py patterns --markdown` — save FEEDBACK-PATTERNS.md to channel-data/patterns/
   - Calls feedback_queries.generate_patterns_report()
   - Markdown saved to: `{project_root}/channel-data/patterns/FEEDBACK-PATTERNS.md`

**CLI help text (epilog):**
```
Examples:
  python feedback.py backfill               Import all analysis files
  python feedback.py backfill --force       Re-import all (overwrite existing)
  python feedback.py query --topic territorial  Show territorial video insights
  python feedback.py query --video VIDEO_ID    Show specific video feedback
  python feedback.py query --topic territorial --metric retention  Filter by metric
  python feedback.py patterns               Show success/failure patterns
  python feedback.py patterns --markdown    Save patterns report
```

**Implementation details:**
- Use `from pathlib import Path` for project root detection (2 levels up from script)

- Import feedback_parser for backfill
- Import feedback_queries for query and patterns

- Handle missing data gracefully: if no feedback stored, print helpful message suggesting `python feedback.py backfill` first
- Exit codes: 0 success, 1 no results, 2 error

**Terminal output formatting:**
- Use `format_query_terminal` / `format_patterns_terminal` from feedback_queries
- For video-specific query, show: video_id, title, topic_type, conversion_rate, drop_point, observations, actionable items
- For topic query, show: table of videos with key metrics, then aggregated insights below
- For patterns, show: success patterns then failure patterns with counts

**Markdown output:**
- For query: full detail with all observations
- For patterns: save to channel-data/patterns/FEEDBACK-PATTERNS.md, print path
